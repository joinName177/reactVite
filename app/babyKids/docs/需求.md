## 1. 需求梳理

### 1.1 应用名称（暂定）
`KidEnglishCards` 或 `宝宝英语卡`

### 1.2 目标用户
- 家长辅助孩子（2-8岁）学习英语
- 孩子可以通过图片和声音认知单词

### 1.3 核心功能点
| 功能 | 描述 |
|------|------|
| **图片展示** | 显示当前单词对应的图片，支持本地图片和用户自定义图片。 |
| **中英文显示** | 图片下方显示英文单词和中文翻译，字体大而清晰。 |
| **语音播放** | 点击“英文”按钮播放英文发音，点击“中文”按钮播放中文发音（TTS）。 |
| **添加新单词** | 用户从手机相册选择图片，输入英文和中文，即可生成新的单词卡片。 |
| **编辑卡片** | 支持修改已有卡片的图片、英文、中文，入口为卡片详情页的“编辑”按钮。 |
| **删除卡片** | 长按卡片或点击“删除”按钮，弹出确认弹窗后删除，同时删除本地存储的图片文件。 |
| **本地存储** | 所有用户添加的单词和图片永久保存在手机本地，重启App后依然存在。 |
| **卡片切换** | 多张卡片时通过“上一个/下一个”按钮浏览；仅一张卡片时按钮隐藏或禁用。支持左右滑动切换（可选）。 |
| **预设示例** | 内置少量单词（如猫、狗）作为示例，方便初次使用。 |
| **空状态引导** | 无卡片时显示引导界面，提示用户添加第一张卡片。 |
| **打包安装** | 最终生成APK文件，可直接安装到Android手机。 |

### 1.4 卡片顺序
- 默认按添加时间倒序（最新添加的在前）。
- 后续可扩展：按字母排序、拖拽自定义顺序。

### 1.5 非功能需求
- 界面简洁友好，按钮大而醒目，适合儿童操作。
- 语音语速较慢（默认 0.8x），发音清晰，可在设置中调节语速（可选）。
- 数据完全离线，不依赖服务器。
- 占用空间小，图片压缩存储（见 2.1.1 图片压缩策略）。
- **最低系统版本**：Android 5.0+；若支持 iOS，则 iOS 12+。
- **无障碍**：大字体、高对比度，便于儿童和家长使用。

### 1.6 页面结构
| 页面 | 说明 |
|------|------|
| **主卡片浏览页** | 展示当前卡片，支持切换、播放语音、进入编辑/删除。 |
| **添加/编辑卡片页** | 选择图片、输入英文和中文，保存或更新卡片。 |
| **设置页**（可选） | 语速调节、关于等。 |

### 1.7 UI/UX 细节
- **卡片切换方式**：以“上一个/下一个”按钮为主；可扩展左右滑动。
- **加载与错误**：图片加载失败时显示占位图或提示；TTS 不可用时提示“语音不可用”；权限被拒时引导用户去设置开启。
- **示例图片来源**：使用 Unsplash、Pexels 等免费可商用图库，或自行拍摄。

## 2. 研发流程与环境搭建

### 2.1 技术选型
- **框架**：React Native + Expo
  - 原因：一套代码生成iOS和Android应用，开发效率高；Expo内置常用API（相册、文件系统、语音），无需配置原生环境。
- **本地存储**：
  - 单词列表：AsyncStorage（键值对存储，适合小数据量）
  - 图片文件：expo-file-system（将图片复制到应用私有目录）
- **语音**：expo-speech
- **图片选择**：expo-image-picker

#### 2.1.1 数据存储结构
- 存储格式版本号：`storageVersion: 1`，用于后续数据迁移兼容。
- 单张卡片 JSON 示例：
```json
{
  "id": "uuid-xxx",
  "english": "cat",
  "chinese": "猫",
  "imageUri": "file:///path/to/image.jpg",
  "createdAt": "2025-03-18T10:00:00.000Z"
}
```
- 完整数据键：`kidEnglishCards`，值为上述对象数组。

#### 2.1.2 图片压缩策略
- 用户选择图片后，保存前进行压缩：最大宽度 800px，JPEG 质量 80%。
- 避免原图过大占用空间，同时保证卡片展示清晰度。

### 2.2 环境准备
#### 2.2.1 安装Node.js
- 官网下载 LTS 版本：https://nodejs.org/
- 安装完成后在终端验证：`node -v` 和 `npm -v`

#### 2.2.2 安装 EAS CLI（用于打包 APK）
```bash
npm install -g eas-cli
```

#### 2.2.3 注册 Expo 账号
- 访问 https://expo.dev 注册一个免费账号
- 在终端登录：`eas login`

#### 2.2.4 手机安装 Expo Go（开发调试用）
- iOS：App Store 搜索 “Expo Go”
- Android：应用商店搜索 “Expo Go”

### 2.3 创建项目
```bash
npx create-expo-app KidEnglishCards -t expo-template-blank-typescript
cd KidEnglishCards
```
> 注：`expo init` 已弃用，推荐使用 `create-expo-app`。

### 2.4 安装依赖
```bash
expo install @react-native-async-storage/async-storage expo-file-system expo-image-picker expo-speech
```

### 2.5 添加预设图片
- 在项目根目录创建 `assets` 文件夹
- 放入示例图片，如 `cat.jpg`, `dog.jpg`（可从 Unsplash、Pexels 等免费图库下载，注意版权）

### 2.6 编写代码
参考之前提供的完整 `App.tsx` 代码（已包含本地存储逻辑）。你可以直接复制并调整图片路径。

**注意**：代码中使用了 `require('./assets/cat.jpg')`，请确保文件名一致。

### 2.7 运行调试
```bash
npx expo start
```
- 用手机Expo Go扫描终端二维码，即可在手机上预览App。
- 测试所有功能：选择图片、输入单词、播放声音、切换卡片、关闭重启验证数据持久化。

### 2.8 打包APK
#### 2.8.1 配置eas.json
项目根目录下运行：
```bash
eas build:configure
```
选择 `Android` 平台，生成 `eas.json` 文件。修改 `preview` 配置以生成APK：
```json
{
  "build": {
    "preview": {
      "android": {
        "buildType": "apk"
      }
    },
    "production": {}
  }
}
```

#### 2.8.2 执行构建
```bash
eas build -p android --profile preview
```
- 云端构建需要一些时间（通常5-10分钟），完成后会提供APK下载链接。
- 将APK下载到手机，点击安装（可能需要开启“允许未知来源应用”）。

### 2.9 发布与分发
- 你可以将APK分享给家人朋友，或者上传到应用商店（如果需要）。
- 如果要上架Google Play，需要生成签名密钥并配置，目前可先直接安装APK使用。

## 3. 关键注意事项

### 3.1 图片权限
- 用户从相册选择图片时，App 会请求权限，需在代码中处理权限拒绝的情况。
- **权限说明文案**：相册权限可写为「用于选择单词卡片的图片」，便于用户理解和应用商店审核。

### 3.2 删除卡片与图片清理
- 删除单词卡片时，需同时调用 `FileSystem.deleteAsync` 删除对应的本地图片文件，避免占用空间。

### 3.3 数据迁移
- 后续升级 App 时，需确保旧数据兼容新版本。在存储中增加 `storageVersion` 字段，升级时根据版本号做迁移逻辑。

### 3.4 国际化
- 目前界面为中文，如果希望支持其他语言，可以引入i18n库。

### 3.5 测试
- 在多种 Android 版本和屏幕尺寸上测试，确保 UI 适配。
- **边界情况**：空输入、超长单词、无卡片时的首次使用流程。

### 3.6 iOS 支持
- 当前以 Android APK 为主；若需支持 iOS，同一套 Expo 代码可打包为 IPA，上架 App Store 需 Apple 开发者账号。

### 3.7 构建回滚
- EAS 构建失败时，可检查 `eas.json` 配置和依赖版本；保留上一版 APK 作为回退。

### 4.1 阶段划分
| 阶段 | 内容 |
|------|------|
| **MVP** | 核心功能：添加卡片、展示、语音、切换、本地存储、预设示例 |

### 4.2 验收标准（示例）
- 添加卡片后重启 App，数据仍存在。
- 删除卡片后，对应图片文件已从本地移除。
- 无卡片时显示引导界面，引导用户添加第一张卡片。
