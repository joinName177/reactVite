# 执行需求工厂完整流程（/pm-flow）

你正在被 **`/pm-flow`** 命令调用。请按「需求工厂完整流程」执行，并遵循项目中的规则与技能。

## 若用户未附带需求描述

先询问用户：
「请提供一句话需求或需求描述；也可直接写在这里，我会按 P0 澄清 → S01-S19 → 原型 → 验收的流程执行。」

## 若用户已附带需求描述

1. **遵循规则**：遵守 `.cursor/rules/product-requirement-flow.mdc` 中的全部要求。
2. **先读技能**：读取并遵循 `.cursor/skills/requirement-factory/SKILL.md` 中的流程与步骤。
3. **第一轮回复**：只输出 **P0 澄清问题清单**（用户与场景、对象与范围、主路径、规则与边界、验收口径），不要跳过直接给方案或代码。
4. **后续步骤**：等用户回复澄清答案后，再进入 S01-S19 分析与方案阶段；用户确认方案后再执行文档与可交互原型；原型须严格按 03 功能清单逐条实现；可选执行自动化验收。
