# 03 功能清单专项审查

对已产出的 03-功能清单与验收标准 做二次审查，调用 product-expert 做功能清单专项审查，降低功能遗漏风险。

## 适用场景

- 人工撰写的 03 需复核
- 从其他流程迁移来的 03 需对齐标准
- 02 变更后需复核的 03
- 需求工厂流程产出的 03 做二次审查

## 输入约定

1. **显式指定**：用户可 `@02-需求方案-xxx.md` 和 `@03-功能清单与验收标准-xxx.md` 显式指定审查对象。
2. **推断**：若未指定，根据当前打开/聚焦的 03 文件推断，并自动查找同模块的 02。
3. **无法推断**：若无法推断，提示用户 @ 指定 02 和 03 文件。

## 执行要求

1. **读取文件**：读取 02 全文 + 03 全文（含自检表）。
2. **调用技能**：调用 `.cursor/skills/requirement-factory/product-expert/SKILL.md` 中的「功能清单专项审查」维度。
3. **输出**：输出审查结论（评审通过 / 问题列表）；每条问题 ≤30 字，总输出 ≤400 字。
4. **可选动作**：主 agent 可根据专家建议直接修正 03，并更新自检表打勾；若专家指出需用户确认的问题，向用户展示并等待确认后再定稿。

## 触发方式

- 用户输入 `/pm-audit-03` 即表示要执行本审查。
- 可附带 `@02-需求方案-xxx.md` 和 `@03-功能清单与验收标准-xxx.md` 指定文件。
