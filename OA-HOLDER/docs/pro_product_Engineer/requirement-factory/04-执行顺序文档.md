# 需求工厂执行顺序文档

> 用于生成 Skills 的标准化执行顺序定义  
> **规则**：每步执行后必须自检，遇到任何问题需停留确认，解决后再进行下一步

---

## 执行规则

1. **严格顺序**：按步骤编号顺序执行，不可跳步
2. **自检必过**：每步自检项全部通过后方可进入下一步
3. **问题停留**：自检不通过或遇到疑问时，暂停执行，与需求方/相关方确认后再继续
4. **输出前置**：下一步依赖上一步的输出物

---

## 步骤定义（用于 Skills 生成）

### Step 01：5W2H 拆解

| 属性 | 内容 |
|------|------|
| **步骤ID** | S01 |
| **步骤名称** | 5W2H 拆解 |
| **前置条件** | 已有一句话需求输入 |
| **输入** | 一句话需求（来自「一句话需求场景.text」） |
| **执行动作** | 对 Who、What、Why、Where、When、How、How much 七个维度逐一回答 |
| **输出** | 5W2H 澄清表（7 个维度均有明确回答） |
| **自检项** | ① 7 个维度均有明确回答 ② 回答与原始需求一致、无矛盾 |
| **问题处理** | 自检不通过 → 暂停 → 与需求方确认 → 修正 → 重新自检 → 通过后进入 S02 |
| **下一步** | S02 |

## 执行顺序流程图

```
S01 5W2H 拆解
    ↓ [自检通过]
S02 需求澄清会
    ↓ [自检通过]
S03 用户画像 → S04 用户旅程 → S05 竞品/现状分析
    ↓ [自检通过]
S06 功能脑暴 → S07 MVP 筛选 → S08 功能优先级
    ↓ [自检通过]
S09 信息架构 → S10 流程图 → S11 状态流转
    ↓ [自检通过]
S12 低保真原型 → S13 交互说明
    ↓ [自检通过]
S14 高保真原型（可选）
    ↓ [自检通过]
S15 原型评审
    ↓ [自检通过]
S16 功能清单输出 → S17 非功能需求
    ↓ [自检通过]
S18 PRD 编写 → S19 交付物确认
    ↓ [自检通过]
交付开发
```

## Skills 生成说明

本文档的步骤定义（步骤ID、步骤名称、前置条件、输入、执行动作、输出、自检项、问题处理、下一步）可直接用于生成 Cursor Skills。每个 Skill 应包含：

1. **触发条件**：用户意图或步骤ID
2. **执行逻辑**：按「执行动作」执行
3. **自检逻辑**：按「自检项」逐项检查
4. **问题处理**：自检不通过时提示用户确认，不自动进入下一步
5. **输出物**：按「输出」定义产出
