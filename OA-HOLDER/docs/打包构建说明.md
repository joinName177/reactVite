# OA-HOLDER 打包构建说明

> 文档版本：1.0  
> 更新日期：2026-03-12

---

## 一、概述

本文档说明 OA-HOLDER 项目的**构建**与**打包**流程，包括开发模式、生产构建、打包命令、输出目录及常见问题。

### 1.1 构建与打包的区别

| 阶段 | 说明 | 产物 |
|------|------|------|
| **构建** | 编译源码为可执行代码 | `dist/main/`、`dist/renderer/` |
| **打包** | 将构建产物打包成安装程序 | `release/v{version}/` 下的 .exe 等 |

---

## 二、构建流程概览

```
┌─────────────────────────────────────────────────────────────────┐
│                        构建流程                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 共享包构建（Turborepo 并行）                                 │
│     @holder/shared-types → @holder/ipc-bridge → @holder/api       │
│     @holder/ui                                                 │
│                          ↓                                      │
│  2. 应用构建                                                    │
│     @holder/shell (Vite)  → dist/renderer/                       │
│     @holder/electron (tsc) → dist/main/                         │
│                          ↓                                      │
│  3. 打包（electron-builder）                                    │
│     dist/ + package.json → release/v{version}/                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 三、开发模式

### 3.1 启动开发环境

```bash
pnpm dev
```

**执行步骤**：
1. 构建主进程（electron）
2. 启动壳应用开发服务器（Vite，端口 5173）
3. 监听主进程变更并自动重新编译
4. 待开发服务器就绪后启动 Electron

### 3.2 仅启动壳应用（浏览器调试）

```bash
pnpm --filter @holder/shell dev
```

仅启动 Vite 开发服务器，不启动 Electron，用于纯前端调试。

### 3.3 仅构建主进程

```bash
pnpm --filter @holder/electron build
```

## 四、构建命令

### 4.1 全量构建（生产）

```bash
pnpm build
```

通过 Turborepo 按依赖顺序构建所有包（shared-types → ipc-bridge/api/ui → shell/electron）。

**等价于**：
```bash
pnpm run build:prod
```

### 4.2 分步构建

```bash
# 1. 构建共享包
pnpm run build:packages

# 2. 构建壳应用（Vite）
pnpm run build:shell

# 3. 构建主进程（tsc）
pnpm run build:electron
```

### 4.3 多环境构建

| 命令 | 说明 |
|------|------|
| `pnpm run build:prod` | 生产环境（production） |
| `pnpm run build:re` | RE 预发布环境 |

### 4.4 构建输出目录

| 路径 | 内容 |
|------|------|
| `dist/main/` | 主进程编译产物（tsc 输出） |
| `dist/renderer/` | 壳应用构建产物（Vite 输出） |
| `packages/shared/*/dist/` | 各共享包编译产物 |

## 五、打包命令

### 5.1 命令速查

| 命令 | 说明 |
|------|------|
| `pnpm run dist` | 生产环境 + 打包（默认 x64） |
| `pnpm run dist:re` | RE 环境 + 打包 |
| `pnpm run dist:win` | 生产环境 + Windows 32位+64位 |
| `pnpm run dist:win32` | 生产环境 + 仅 Windows 32位 |
| `pnpm run dist:win64` | 生产环境 + 仅 Windows 64位 |
| `pnpm run dist:re:win` | RE 环境 + Windows 32位+64位 |
| `pnpm run dist:re:win32` | RE 环境 + 仅 Windows 32位 |
| `pnpm run dist:re:win64` | RE 环境 + 仅 Windows 64位 |
| `pnpm run pack` | 仅打包（不执行构建，需已构建） |

### 5.2 打包流程

以 `pnpm run dist` 为例：

```
1. pnpm run build:prod     # 构建生产版本
2. node scripts/build-with-version.js   # 按版本号打包
```

### 5.3 输出目录结构

打包产物按版本号输出到 `release/v{version}/`：

```
release/
├── v1.0.5/                           # 版本 1.0.5
│   ├── holder Setup 1.0.5-x64.exe    # NSIS 安装程序（64位）
│   ├── holder Setup 1.0.5-ia32.exe   # NSIS 安装程序（32位，可选）
│   ├── holder 1.0.5-x64.exe          # 便携版（64位，可选）
│   ├── holder 1.0.5-ia32.exe         # 便携版（32位，可选）
│   ├── win-unpacked/                  # 解压目录（64位）
│   ├── win-ia32-unpacked/             # 解压目录（32位）
│   ├── latest.yml                    # 自动更新检查文件
│   └── *.exe.blockmap                 # 增量更新文件（可选）
├── v1.0.6/
└── ...
```

## 六、版本管理

### 6.1 版本号位置

在根目录 `package.json` 中的 `version` 字段：

```json
{
  "version": "1.0.5"
}
```

### 6.2 更新版本号

1. 编辑 `package.json`，修改 `version`
2. 执行 `pnpm run dist`，打包脚本会自动创建 `release/v{version}/` 目录

### 6.3 版本号规范

建议遵循 [语义化版本](https://semver.org/)：

- **主版本号**：不兼容的 API 修改
- **次版本号**：向下兼容的功能新增
- **修订号**：向下兼容的问题修正

### 7.1 图标

| 文件 | 用途 |

| 目录 | 说明 |
|------|------|
| `build/` | 构建资源（图标等） |
| `dist/` | 构建输出（构建产物） |
| `release/` | 打包输出（安装程序、便携版等） |

## 九、常见问题

### 9.1 打包时提示找不到图标

**解决方案**：在 `build/` 目录下放置 `icon.ico`，或从 `package.json` 的 `build.win.icon` 中移除图标配置。

### 9.2 打包文件过大

**解决方案**：
- 检查 `package.json` 中 `build.files`，排除无用文件
- 参考 [构建优化说明.md](./构建优化说明.md)

### 9.3 打包速度慢

**解决方案**：
- 参考 [构建优化说明.md](./构建优化说明.md)
- 仅打包单一架构：`pnpm run dist:win64` 替代 `dist:win`

### 9.4 32 位打包失败

**解决方案**：部分 Electron 版本可能不再支持 32 位，可尝试仅打包 64 位：`pnpm run dist:win64`。

### 9.5 仅生成解压目录（不打包安装程序）

```bash
pnpm run build:prod
npx electron-builder --win --x64 --dir
```

### 9.6 临时配置文件未删除

**现象**：`electron-builder.temp.json` 残留。

**解决方案**：手动删除 `electron-builder.temp.json`，下次打包会重新生成并清理。

## 十、打包脚本说明

`scripts/build-with-version.js` 实现按版本号输出：

1. 读取 `package.json` 的 `version`
2. 创建 `release/v{version}/` 输出目录
3. 通过临时配置传递给 electron-builder
4. 打包完成后删除临时文件

自定义输出格式：编辑脚本中的 `const outputDir = \`release/v${version}\``
