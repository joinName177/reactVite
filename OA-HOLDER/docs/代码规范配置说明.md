# 代码规范配置说明

## 概述

项目已配置 ESLint 和 Prettier，遵循最规范的 React 代码规范。

## 配置说明

### Prettier 配置 (`.prettierrc`)

- `singleAttributePerLine: false` - 允许属性在同一行（适合自闭合标签如 img、input）
- `printWidth: 100` - 每行最大字符数
- `jsxSingleQuote: true` - JSX 中使用单引号

### ESLint 配置 (`.eslintrc.cjs`)

#### React 规则

- `react/jsx-max-props-per-line` - 单行最多 5 个属性（适合自闭合标签）
- `react/jsx-first-prop-new-line: never` - 第一个属性不换行
- `react/react-in-jsx-scope: off` - React 17+ 不需要导入 React

#### TypeScript 规则

- 完整的 TypeScript 类型检查
- 未使用变量警告

## 代码格式规范

### ✅ 正确：自闭合标签保持单行

```tsx
// img 标签
<img src={icon} alt='图标' />

// input 标签
<input type='text' placeholder='请输入' />

// 在 JSX 属性中使用
<Input prefix={<img src={icon} alt='图标' />} />
```

### ❌ 错误：自闭合标签换行

```tsx
// 不推荐
<img
  src={icon}
  alt='图标'
/>

// 不推荐
<Input
  prefix={
    <img
      src={icon}
      alt='图标'
    />
  }
/>
```

### ✅ 正确：多属性组件可以换行

```tsx
// 当属性较多时，可以换行
<Button type='primary' size='large' onClick={handleClick} disabled={loading}>
  提交
</Button>
```

## 使用方式

### 格式化代码

```bash
# 使用 Prettier 格式化
npx prettier --write "src/**/*.{ts,tsx,js,jsx}"

# 使用 ESLint 检查和修复
npx eslint "src/**/*.{ts,tsx,js,jsx}" --fix
```

### VS Code 配置

在 `.vscode/settings.json` 中添加：

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

## 已安装的包

- `eslint` - ESLint 核心
- `@typescript-eslint/parser` - TypeScript 解析器
- `@typescript-eslint/eslint-plugin` - TypeScript 插件
- `eslint-plugin-react` - React 插件
- `eslint-plugin-react-hooks` - React Hooks 插件
- `eslint-plugin-jsx-a11y` - JSX 可访问性插件
- `eslint-config-prettier` - 禁用与 Prettier 冲突的规则
