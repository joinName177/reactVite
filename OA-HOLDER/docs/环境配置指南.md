# 环境配置指南

> 整合自：环境配置说明、环境配置整合说明、环境配置使用示例

---

## 一、概述

项目支持三套环境：**开发**（development）、**RE 预发布**（re）、**生产**（production），通过 `VITE_APP_ENV` 与 `.env.*` 文件控制。

---

## 二、环境文件

| 文件 | 用途 |
|------|------|
| `.env.development` | 开发/测试环境 |
| `.env.re` | RE 预发布环境 |
| `.env.production` | 生产环境 |
| `.env.local` | 本地覆盖（已加入 .gitignore，不提交） |

---

## 三、环境变量

### 3.1 命名规则

所有在渲染进程中使用的变量必须以 `VITE_` 开头。

### 3.2 常用变量

| 变量 | 说明 |
|------|------|
| `VITE_APP_ENV` | 环境标识（development/re/production） |
| `VITE_APP_TITLE` | 应用标题 |
| `VITE_API_BASE_URL` | API 基础地址 |
| `VITE_WS_BASE_URL` | WebSocket 地址 |
| `VITE_DEBUG` | 是否开启调试（true/false） |

### 3.3 添加新变量

1. 在对应 `.env.*` 中添加：`VITE_NEW_VAR=value`
2. 在 `apps/shell/src/types/env.d.ts` 中补充类型声明

---

## 四、构建命令

| 环境 | 构建 | 打包 |
|------|------|------|
| 开发 | `pnpm dev` | — |
| RE | `pnpm run build:re` | `pnpm run dist:re` |
| 生产 | `pnpm run build:prod` | `pnpm run dist` |

## 五、代码中使用

### 5.1 导入配置

```typescript
import envConfig, { isDevelopment, isRe, isProduction } from '@/config/env';
```

### 5.2 访问配置

```typescript
// 便捷属性
const apiUrl = envConfig.apiBaseUrl;
const wsUrl = envConfig.wsBaseUrl;
const title = envConfig.title;
const debug = envConfig.debug;

// 环境判断
if (isDevelopment()) {
  console.log('开发环境');
}
if (isRe()) {
  console.log('RE 环境');
}
if (isProduction()) {
  console.log('生产环境');
}
```

### 5.3 直接使用 Vite 变量

```typescript
const apiUrl = import.meta.env.VITE_API_BASE_URL;
const isDev = import.meta.env.VITE_APP_ENV === 'development';
```

### 5.4 示例：API 请求

```typescript
import envConfig from '@/config/env';

const response = await fetch(`${envConfig.apiBaseUrl}/api/users`, {
  method: 'GET',
  headers: { Authorization: `Bearer ${token}` },
});
```

### 5.5 示例：条件渲染

```typescript
import { isDevelopment } from '@/config/env';

const MyComponent = () => (
  <div>
    {isDevelopment() && <div>调试信息</div>}
    <div>正常内容</div>
  </div>
);
```

---

## 六、注意事项

1. **修改后重启**：修改环境变量后需重启开发服务器或重新构建
2. **敏感信息**：不要将密钥等提交到仓库，使用 `.env.local`
3. **RE 环境**：Vite 默认只支持 development/production，RE 需额外配置（如 `--mode re`）
