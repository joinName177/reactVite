## 一、项目概述

### 1.1 项目简介

OA-HOLDER 是一款基于 Electron 的桌面办公应用，采用 **单包（Single Package）** 架构，整合工作台、任务管理、沟通协作、审批、专项管理、绩效、OKR 等企业 OA 核心功能。

### 1.2 架构特点

| 特点 | 说明 |
|------|------|
| **单包架构** | 单一 package.json，所有代码统一管理，无 workspace 依赖 |
| **分层目录** | 主进程 / 渲染进程 / 共享模块 通过目录划分，逻辑清晰 |
| **类型安全** | 端到端 TypeScript，IPC 通讯具备类型契约 |
| **模块化** | src/shared 下按职责划分模块，便于维护与扩展 |

### 1.3 技术栈总览

| 层级 | 技术选型 | 版本 |
|------|----------|------|
| 运行时 | Electron | 39.x |
| 前端框架 | React | 19.x |
| 构建工具 | Vite | 6.x |
| 语言 | TypeScript | 5.9 |
| UI 组件库 | Ant Design | 6.x |
| 状态管理 | Zustand | 5.x |
| 路由 | React Router | 7.x |
| 图标 | Lucide React | 0.47x |
| 包管理 | pnpm | 9.14 |

## 二、目录结构

### 2.1 根目录结构

```
OA-HOLDER/
├── src/                     # 源码
│   ├── main/                # Electron 主进程
│   ├── renderer/            # 渲染进程（壳应用）
│   └── shared/              # 共享模块（类型、API、UI、IPC 等）
├── docs/                    # 项目文档
├── build/                   # 构建资源（图标等）
├── dist/                    # 构建输出
├── release/                 # 打包产物
├── scripts/                 # 构建脚本
├── package.json             # 唯一包配置
├── tsconfig.json            # 渲染进程 TS 配置
├── tsconfig.main.json       # 主进程 TS 配置
├── vite.config.ts           # Vite 构建配置
├── biome.json               # 代码风格配置（Biome 替代 ESLint/Prettier）
├── .editorconfig            # 编辑器配置（统一 UTF-8、换行等）
├── .env.development         # 开发环境变量
├── .env.production          # 生产环境变量
└── .env.re                  # RE 环境变量
```

### 2.2 主进程（src/main/）

```
src/main/
├── index.ts                 # 主进程入口，启动 Electron
├── main.ts                  # 主逻辑编排，初始化各模块
├── core/                    # 应用核心逻辑
│   ├── app-lifecycle.ts     # 应用生命周期（启动/退出/就绪）
│   ├── single-instance.ts   # 单实例锁，防止多开
│   └── config.ts            # 主进程配置（窗口尺寸、URL 等）
├── windows/                 # 窗口管理
│   ├── window-manager.ts    # 窗口创建、销毁、聚焦
│   ├── window-pool.ts       # 窗口池，预创建/复用窗口
│   └── window-config.ts     # 窗口配置（主窗口/模块窗口）
├── ipc/                     # 进程间通讯
│   ├── ipc-hub.ts           # IPC 总线，注册与分发 handler
│   └── shared-state.ts      # 跨窗口共享状态中心
├── native/                  # 原生能力封装
│   ├── tray.ts              # 系统托盘图标与菜单
│   └── auto-update.ts       # 自动更新逻辑
├── preload/                 # 预加载脚本（注入渲染进程）
│   ├── main-preload.ts      # 主窗口 preload，暴露安全 API
│   └── module-preload.ts    # 独立模块窗口 preload
└── utils/                   # 工具函数
    └── machine-id.ts        # 机器标识
```

**职责**：应用生命周期、窗口管理、IPC 总线、系统托盘、自动更新、预加载脚本。

### 2.3 渲染进程（src/renderer/）

```
src/renderer/
├── main.tsx                 # 渲染进程入口
├── app.tsx                  # 根组件（I18nProvider + ConfigProvider + Router）
├── index.html               # HTML 入口
├── layout/                  # 布局组件
│   ├── app-layout.tsx       # 主布局（侧边栏 + 顶栏 + 内容区）
│   ├── sidebar/             # 侧边栏导航菜单（含 locale）
│   └── header/              # 顶栏（用户、通知、设置，含 locale）
├── router/                  # 路由配置
│   ├── index.tsx            # 路由定义、懒加载、ProtectedRoute
│   └── locale.ts           # 路由相关国际化
├── modules/                 # 内嵌业务模块
│   ├── login/               # 登录页（账号/扫码/忘记密码）
│   ├── workdesk/            # 工作台
│   ├── task-manager/        # 任务管理（占位）
│   ├── settings/            # 设置
│   └── not-found/           # 404 页面
├── components/              # 渲染进程级组件
│   └── protected-route.tsx  # 路由守卫（未登录跳转 /login）
├── store/                   # 全局状态（Zustand）
│   ├── user.store.ts        # 用户信息、登录状态
│   ├── ui.store.ts          # UI 状态（环境、侧边栏等）
│   └── notification.store.ts # 通知数量、未读状态
├── hooks/                   # 通用 Hooks
│   └── use-module.ts        # 模块加载 Hook
├── config/                  # 环境与运行配置

├── assets/                  # 静态资源（图标、图片等）
├── styles/                  # 全局样式
└── types/                   # 类型声明（env.d、electron.d）
```

**职责**：主窗口 UI、侧边栏导航、顶栏、核心模块路由、全局状态。

## 四、核心模块说明

### 4.1 主进程（Electron）

| 模块 | 文件 | 说明 |
|------|------|------|
| 应用核心 | core/app-lifecycle.ts | 启动、退出、单例锁 |
| 窗口管理 | windows/window-manager.ts | 创建、销毁、聚焦窗口 |
| 窗口池 | windows/window-pool.ts | 预创建/复用窗口 |
| IPC 总线 | ipc/ipc-hub.ts | 注册与分发 IPC 处理器 |
| 共享状态 | ipc/shared-state.ts | 跨窗口状态同步 |
| 托盘 | native/tray.ts | 系统托盘菜单 |
| 自动更新 | native/auto-update.ts | 应用更新逻辑 |

### 4.2 壳应用（Shell）

| 模块 | 说明 |
|------|------|
| 侧边栏 | 主导航菜单，路由到各功能模块 |
| 顶栏 | 用户信息、通知、设置入口 |
| 路由 | React Router 7，按路径加载页面 |
| 状态 | Zustand：用户、UI、通知等全局状态 |

### 4.3 IPC 通讯

主进程与渲染进程通过 `src/shared/ipc-bridge` 进行类型安全通讯：

- **主进程**：在 ipc-hub 中注册 handler
- **渲染进程**：通过 preload 暴露的 API 调用
- **类型契约**：在 `src/shared/types` 中定义 IPC 消息格式

## 五、构建与开发

### 5.1 构建流程

```
pnpm install
    │
    ├── 构建主进程（tsc → dist/main）
    │
    └── 构建渲染进程（Vite → dist/renderer）
```

> 单包架构下，shared 模块作为源码直接参与构建，无需单独编译。

### 5.2 开发命令

| 命令 | 说明 |
|------|------|
| `pnpm dev` | 构建主进程 + 启动渲染进程开发服务器 + 主进程 watch + 启动 Electron |
| `pnpm run dev:renderer` | 仅启动渲染进程开发服务器（Vite，端口 5173） |
| `pnpm run dev:main:watch` | 主进程 TypeScript 监听编译 |
| `pnpm run dev:electron:start` | 启动 Electron |
| `pnpm run build` | 构建主进程 + 渲染进程 |
| `pnpm run build:main` | 仅构建主进程（tsc + prepare-electron-main） |
| `pnpm run build:renderer` | 仅构建渲染进程（Vite production） |
| `pnpm run build:re` | RE 环境构建 |
| `pnpm run dist` | 全量构建 + electron-builder 打包 |
| `pnpm run lint` | Biome 代码检查 |
| `pnpm run format` | Biome 格式化 |

### 5.3 输出目录

| 路径 | 内容 |
|------|------|
| dist/main/ | 主进程编译产物 |
| dist/renderer/ | 壳应用构建产物 |
| release/ | electron-builder 打包输出 |

## 六、数据流与状态管理

### 6.1 状态分层

| 层级 | 方案 | 范围 | 示例 |
|------|------|------|------|
| 跨窗口共享 | IPC + SharedState | 主进程维护 | 用户信息、Token、通知计数 |
| 服务端数据 | API 请求 | 按需拉取 | 任务列表、审批列表 |
| 模块级 UI | Zustand | Shell 内 | 侧边栏折叠、主题、筛选条件 |
| 组件级 | useState | 组件内 | 输入值、弹窗开关 |

### 6.2 数据流向

```
用户操作 → 渲染进程组件
    │
    ├── 调用 shared/api → 后端 API
    │
    └── 调用 shared/ipc-bridge → 主进程
            │
            └── 主进程处理（窗口、托盘、数据库等）
```

## 九、架构设计收益

本架构设计带来的主要收益如下：

### 9.1 开发效率

| 收益 | 说明 |
|------|------|
| **快速启动** | Vite 按需编译，开发启动 < 3s，HMR < 200ms |
| **结构简单** | 单包架构，无需 workspace、turbo 等配置，上手快 |
| **依赖统一** | 所有依赖集中在根 package.json，版本管理简单 |

### 9.2 可维护性

| 收益 | 说明 |
|------|------|
| **职责清晰** | 主进程 / 渲染进程 / 共享模块 通过目录分层，结构一目了然 |
| **类型安全** | 端到端 TypeScript，IPC 契约统一在 shared/types，减少运行时错误 |
| **引用直观** | 通过路径别名（如 `@/shared/types`）引用，无需 workspace 协议 |

### 9.3 可扩展性

| 收益 | 说明 |
|------|------|
| **模块化** | 业务模块在 src/renderer/modules 下按目录划分，支持懒加载 |
| **共享复用** | UI、API、IPC 等抽成 shared 模块，渲染进程与主进程共同引用 |
| **易于扩展** | 新增模块只需添加目录并注册路由，无需新建包或修改 workspace |

### 9.4 运行与安全

| 收益 | 说明 |
|------|------|
| **进程隔离** | 主进程与渲染进程分离，独立模块可单独窗口，崩溃隔离 |
| **上下文隔离** | preload 暴露最小 API，降低 XSS 等安全风险 |
| **现代运行时** | Electron 39 + Node 22，安全补丁及时、性能更好 |

### 9.5 团队协作

| 收益 | 说明 |
|------|------|
| **统一规范** | 单一项目结构，Biome 配置统一，新人上手更快 |
| **构建简单** | 一次 `pnpm build` 完成全量构建，无多包依赖顺序问题 |
| **文档集中** | 架构文档与代码结构一致，便于沟通与交接 |
