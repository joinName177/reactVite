# 构建优化说明

> 整合自：打包速度优化说明、打包体积优化说明

---

## 一、速度优化

### 1.1 已完成的优化

| 优化项 | 配置 | 效果 |
|--------|------|------|
| Vite 压缩 | `minify: 'esbuild'` | 比 terser 快 10-100 倍 |
| 压缩大小报告 | `reportCompressedSize: false` | 跳过计算，加快构建 |
| Source Map | `sourcemap: false` | 减少构建时间 |
| TypeScript | `incremental: true` | 增量编译，提升 50-80% |
| 压缩级别 | `compression: "normal"` | 比 maximum 快 2-3 倍 |
| 打包目标 | 仅 nsis x64 | 不打包便携版，节省一半时间 |
| 差分包 | `differentialPackage: false` | 禁用差分包，加快打包 |

### 1.2 进一步建议

- **只打包需要的架构**：`pnpm run dist:win64` 替代 `dist:win`
- **CI/CD 缓存**：缓存 `node_modules`、`dist`、Electron 缓存目录
- **Turborepo**：项目已使用，支持并行与增量构建

### 1.3 时间参考

| 阶段 | 优化前 | 优化后 |
|------|--------|--------|
| 首次打包 | ~3-5 分钟 | ~1-2 分钟 |
| 增量打包 | ~2-3 分钟 | ~30-60 秒 |

## 二、体积优化

### 2.1 问题分析

体积过大的常见原因：

- 同时打包 x64 + ia32
- 未排除源码、文档、node_modules
- 包含 source map
- 压缩级别较低

### 2.2 已完成的优化

| 优化项 | 配置 |
|--------|------|
| 排除文件 | `!**/src/**/*`、`!**/docs/**/*`、`!**/*.map` 等 |
| asar 打包 | `asar: true` |
| 仅 x64 | `arch: ["x64"]` |

### 2.3 进一步建议

- **只打包 64 位**：`pnpm run dist:win64`
- **检查依赖**：使用 `npx bundle-phobia [package]` 分析
- **图片优化**：WebP、压缩、CDN

### 2.4 预期体积

- **单个 x64 安装包**：约 120-200 MB（含 Electron 运行时）
- **异常情况**：900MB+ 通常表示配置有误（未排除文件、双架构等）

---

## 三、速度与体积的权衡

| 配置 | 速度 | 体积 |
|------|------|------|
| `compression: "normal"` | 快 | 略大 |
| `compression: "maximum"` | 慢 2-3 倍 | 更小 |
| 仅 x64 | 快 | 小 |
| x64 + ia32 | 慢 | 大 |

当前默认：`compression: "normal"` + 仅 x64，兼顾速度与体积。
