# Build 目录说明

## build/ 目录

此目录用于存放构建相关的资源文件。

## 图标文件

### Windows
- `icon.ico` - Windows 应用程序图标（推荐尺寸：256x256 或 512x512）

### macOS（可选）
- `icon.icns` - macOS 应用程序图标

### Linux（可选）
- `icon.png` - Linux 应用程序图标（推荐尺寸：512x512）

## 如何生成图标

1. 准备一张正方形图片（推荐 512x512 或 1024x1024）
2. 使用在线工具转换为 .ico 格式：
   - https://icoconvert.com/
   - https://cloudconvert.com/png-to-ico
3. 将生成的 `icon.ico` 文件放置在此目录

## 注意事项

- 如果没有图标文件，electron-builder 会使用默认图标
- 图标文件不是必需的，但建议提供以获得更好的用户体验

## release/ 目录

此目录用于存放打包后的应用程序文件。

### 目录结构

打包文件会根据版本号自动组织到对应的文件夹中：

```
release/
├── v1.0.0/              # 版本 1.0.0 的打包文件
├── v1.0.1/              # 版本 1.0.1 的打包文件
└── v1.1.0/              # 版本 1.1.0 的打包文件
```

### 版本文件夹内容

每个版本文件夹包含该版本的所有打包文件：

- **安装程序**：NSIS 安装程序（.exe）
- **便携版**：便携版可执行文件（.exe）
- **解压目录**：未打包的应用文件（用于测试）
- **更新文件**：latest.yml、blockmap 等

### 版本管理优势

- ✅ 每个版本的打包文件独立存放
- ✅ 避免版本文件覆盖
- ✅ 便于版本管理和历史追溯
- ✅ 方便分发特定版本

### 注意事项

- `release/` 目录已添加到 `.gitignore`，不会被提交到版本控制
- 打包时会自动根据 `package.json` 中的版本号创建文件夹
- 旧版本的文件夹不会被自动删除，需要手动清理
