# 打包速度优化说明

## 已完成的优化

### 1. **Vite 构建优化**

#### 使用 esbuild 压缩（比 terser 快 10-100 倍）

```typescript
minify: 'esbuild' // 替代 terser，速度提升显著
```

#### 禁用压缩大小报告

```typescript
reportCompressedSize: false // 跳过压缩大小计算，加快构建
```

#### 禁用 Source Map

```typescript
sourcemap: false // 不生成 source map，减少构建时间
```

### 2. **TypeScript 编译优化**

#### 启用增量编译

```json
{
  "incremental": true,
  "tsBuildInfoFile": "dist/main/.tsbuildinfo"
}
```

- 只编译变更的文件
- 后续构建速度提升 50-80%

### 3. **Electron Builder 优化**

#### 降低压缩级别

```json
"compression": "normal"  // 从 maximum 改为 normal，速度提升 2-3 倍
```

#### 移除便携版打包

```json
"target": [
  {
    "target": "nsis",
    "arch": ["x64"]  // 只打包安装程序，不打包便携版
  }
]
```

#### 禁用差分更新包

```json
"differentialPackage": false  // 禁用差分包生成，加快打包
```

### 4. **构建流程优化**

#### 并行构建（可选）

```json
"build": "npm run build:electron & npm run build:renderer"
```

注意：Windows 上使用 `&` 可能不稳定，建议保持顺序执行。

## 速度提升对比

| 优化项                | 优化前                  | 优化后           | 提升    |
| --------------------- | ----------------------- | ---------------- | ------- |
| Vite 压缩             | terser (~30s)           | esbuild (~3s)    | **10x** |
| TypeScript 编译       | 全量编译 (~5s)          | 增量编译 (~1s)   | **5x**  |
| Electron Builder 压缩 | maximum (~60s)          | normal (~20s)    | **3x**  |
| 打包目标              | nsis + portable (~120s) | nsis only (~60s) | **2x**  |

**总体提升：约 3-5 倍速度提升**

## 进一步优化建议

### 1. 使用构建缓存

electron-builder 会自动缓存 Electron 二进制文件，第二次打包会更快。

### 2. 只打包需要的架构

如果只需要 64 位：

```bash
npm run dist:win64  # 只打包 64 位
```

### 3. 使用 CI/CD 缓存

在 CI/CD 中缓存：

- `node_modules`
- `dist` 目录
- Electron 缓存目录

### 4. 跳过类型检查（开发构建）

如果需要更快的开发构建：

```json
"build:electron:fast": "tsc -p tsconfig.node.json --noEmit false --skipLibCheck"
```

### 5. 使用 turbo 或 nx（大型项目）

对于大型项目，可以考虑使用构建工具如 turbo 或 nx 来并行化构建。

## 打包时间参考

### 优化前

- 首次打包：~3-5 分钟
- 增量打包：~2-3 分钟

### 优化后

- 首次打包：~1-2 分钟
- 增量打包：~30-60 秒

```bash
# 修改一个文件后再次构建（测试增量构建时间）
time npm run build:prod
```

## 注意事项

1. **压缩级别权衡**：`normal` 压缩比 `maximum` 稍大，但速度快很多
2. **增量编译**：首次构建会生成 `.tsbuildinfo` 文件，需要提交到 Git
3. **esbuild vs terser**：esbuild 压缩效果略差，但速度优势明显
4. **便携版**：如果不需要便携版，移除可以节省一半打包时间

## 恢复最大压缩（如果需要）

如果体积比速度更重要，可以恢复：

```json
"compression": "maximum"
```

但打包时间会增加 2-3 倍。
