# 构建和打包指南

> 本文档与 [项目架构设计文档](./项目架构设计文档.md) 对应，适用于 Monorepo 架构。  
> 项目使用 **pnpm** 作为包管理器。

---

## 一、架构说明

项目采用 Monorepo 架构，构建流程为：

```
共享包（shared-types → ipc-bridge/api/ui）→ 壳应用（shell）→ 主进程（electron）
```

- **构建产物**：`dist/main/`（主进程）、`dist/renderer/`（壳应用）
- **打包产物**：`release/v{version}/`（按版本号输出）

---

## 二、构建命令

### 2.1 开发模式

```bash
pnpm dev
```

**执行步骤**：
1. 构建主进程（`@holder/electron`）
2. 启动壳应用开发服务器（Vite，端口 5173）
3. 监听主进程变更并自动重新编译
4. 待开发服务器就绪后启动 Electron

### 2.2 构建生产版本

```bash
pnpm build
```

通过 Turborepo 按依赖顺序构建所有包（共享包 → shell → electron），**不打包**。

### 2.3 分步构建（可选）

```bash
# 1. 构建共享包
pnpm run build:packages

# 2. 构建壳应用
pnpm run build:shell

# 3. 构建主进程
pnpm run build:electron
```

### 2.4 多环境构建

| 命令 | 说明 |
|------|------|
| `pnpm run build:prod` | 生产环境（production） |
| `pnpm run build:re` | RE 预发布环境 |

## 三、打包命令

### 3.1 命令速查

| 命令 | 说明 |
|------|------|
| `pnpm run dist` | 生产环境 + 打包（**默认仅 x64**） |
| `pnpm run dist:re` | RE 环境 + 打包 |
| `pnpm run dist:win` | 生产环境 + Windows 32位+64位 |
| `pnpm run dist:win32` | 生产环境 + 仅 Windows 32位 |
| `pnpm run dist:win64` | 生产环境 + 仅 Windows 64位 |
| `pnpm run dist:re:win` | RE 环境 + Windows 32位+64位 |
| `pnpm run dist:re:win32` | RE 环境 + 仅 Windows 32位 |
| `pnpm run dist:re:win64` | RE 环境 + 仅 Windows 64位 |
| `pnpm run pack` | **仅打包**（不执行构建，需已构建） |

### 3.2 打包流程说明

以 `pnpm run dist` 为例：

1. `pnpm run build:prod` — 构建生产版本
2. `node scripts/build-with-version.js` — 按版本号打包到 `release/v{version}/`

### 3.3 默认打包架构

当前 `package.json` 中 `build.win.target` 默认仅配置 **x64**，因此：

- `pnpm run dist` → 仅生成 64 位安装包
- 需 32 位或双架构时，使用 `dist:win32` 或 `dist:win`

## 四、输出文件说明

### 4.1 版本号文件夹管理

打包完成后，文件会**自动根据版本号**输出到 `release/v{version}/` 目录。

**目录结构示例：**
```
release/
├── v1.0.5/                           # 版本 1.0.5
│   ├── holder Setup 1.0.5-x64.exe    # NSIS 安装程序（64位）
│   ├── holder Setup 1.0.5-ia32.exe   # NSIS 安装程序（32位，dist:win 时）
│   ├── win-unpacked/                  # 解压目录（64位）
│   ├── win-ia32-unpacked/             # 解压目录（32位，dist:win 时）
│   ├── latest.yml                    # 自动更新检查文件
│   └── *.exe.blockmap                 # 增量更新文件（若启用）
├── v1.0.6/
└── ...
```

**说明**：
- 每次打包会**清理**当前版本目录后重新生成
- 不同版本存放在不同文件夹，互不覆盖

### 4.2 文件说明

| 文件/目录 | 说明 |
|-----------|------|
| `holder Setup x.x.x-x64.exe` | NSIS 安装程序（64位） |
| `holder Setup x.x.x-ia32.exe` | NSIS 安装程序（32位） |
| `win-unpacked/` | 64位解压目录（用于测试） |
| `win-ia32-unpacked/` | 32位解压目录 |
| `latest.yml` | 自动更新检查文件 |

**注意**：当前配置默认**不生成便携版**（portable），如需便携版需修改 `build.win.target`。

## 五、图标配置

### 5.1 图标文件

| 文件 | 用途 |
|------|------|
| `build/icon.ico` | Windows 应用图标（推荐 256×256 或 512×512） |
| `build/icon.png` | 通用图标（可选） |

### 5.2 生成工具

- [ICO Convert](https://icoconvert.com/)
- [CloudConvert](https://cloudconvert.com/png-to-ico)
- [RealWorld Icon Editor](http://www.rw-designer.com/)

### 5.3 要求

- **格式**：`.ico` 文件
- **尺寸**：建议 256×256 或 512×512 像素
- **位置**：`build/icon.ico`

若无图标文件，electron-builder 会使用默认图标。

## 七、版本号管理

### 7.1 版本号位置

在根目录 `package.json` 的 `version` 字段：

```json
{
  "version": "1.0.5"
}
```

### 7.2 更新版本号

1. 编辑 `package.json`，修改 `version`
2. 执行 `pnpm run dist`，打包脚本会自动创建 `release/v{version}/` 目录

### 7.3 版本号规范

建议遵循 [语义化版本](https://semver.org/)：`主版本.次版本.修订号`

## 八、常见问题

### 8.1 打包时提示找不到图标

**解决方案**：在 `build/` 目录下放置 `icon.ico`，或从 `package.json` 的 `build.win.icon` 中移除图标配置。

### 8.2 打包文件过大

**解决方案**：
- 检查 `package.json` 中 `build.files`，排除无用文件
- 参考 [构建优化说明.md](./构建优化说明.md)

### 8.3 32 位打包失败

**解决方案**：部分 Electron 版本可能不再支持 32 位，可尝试仅打包 64 位：`pnpm run dist:win64`。

### 8.4 打包速度慢

**解决方案**：参考 [构建优化说明.md](./构建优化说明.md)。

## 九、高级配置

### 9.1 只生成解压目录（不打包安装程序）

```bash
pnpm run build:prod
npx electron-builder --win --x64 --dir
```

### 9.2 自定义输出目录格式

项目已配置为按版本号创建输出文件夹。如需修改格式，编辑 `scripts/build-with-version.js`：

```javascript
// 默认格式：release/v{version}
const outputDir = `release/v${version}`;
```

### 9.3 排除文件

在 `package.json` 的 `build.files` 中配置，当前已排除：`node_modules`、`*.map`、`src`、`docs`、`scripts` 等。
