# 打包体积优化说明

## 问题分析

打包体积过大（900+ MB）的主要原因：

1. **同时打包多个架构**：x64 和 ia32 各生成一个安装包
2. **Electron 运行时**：每个 Electron 应用都包含完整的 Chromium 和 Node.js（约 100-200MB）
3. **未排除不必要的文件**：可能打包了源码、文档等
4. **未启用压缩**：默认压缩级别较低
5. **包含 source map**：生产环境不应该包含

## 已优化的配置

### 1. Electron Builder 配置优化

#### 排除不必要的文件

```json
"files": [
  "dist/**/*",
  "package.json",
  "!**/node_modules/**/*",      // 排除 node_modules
  "!**/*.map",                  // 排除 source map
  "!**/src/**/*",               // 排除源码
  "!**/docs/**/*",              // 排除文档
  "!**/scripts/**/*",           // 排除脚本
  "!**/*.md"                    // 排除 Markdown 文件
]
```

#### 启用压缩

```json
"compression": "maximum",       // 最大压缩
"asar": true,                   // 使用 asar 打包（更高效）
"nsis": {
  "compression": "maximum",     // NSIS 安装包压缩
  "differentialPackage": true   // 差分更新包
}
```

#### 只打包 x64 架构（推荐）

```json
"win": {
  "target": [
    {
      "target": "nsis",
      "arch": ["x64"]  // 只打包 64 位，减少一半体积
    }
  ]
}
```

### 2. Vite 构建优化

#### 禁用 Source Map

```typescript
build: {
  sourcemap: false,  // 生产环境不生成 source map
}
```

#### 启用代码压缩和优化

```typescript
build: {
  minify: 'terser',
  terserOptions: {
    compress: {
      drop_console: true,    // 移除 console
      drop_debugger: true    // 移除 debugger
    }
  }
}
```

#### 代码分割优化

```typescript
rollupOptions: {
  output: {
    manualChunks: {
      'react-vendor': ['react', 'react-dom', 'react-router-dom'],
      'antd-vendor': ['antd', '@ant-design/icons'],
      'redux-vendor': ['@reduxjs/toolkit', 'react-redux']
    }
  }
}
```

## 预期效果

优化后的打包体积：

- **单个 x64 安装包**：约 120-150 MB（包含 Electron 运行时）
- **便携版（portable）**：约 100-130 MB
- **相比之前减少**：约 70-80%

## 进一步优化建议

### 1. 如果不需要 32 位支持

只打包 64 位版本，可以节省一半体积：

```json
"arch": ["x64"]  // 而不是 ["x64", "ia32"]
```

### 2. 使用 Electron 缓存

electron-builder 会自动缓存 Electron 二进制文件，第二次打包会更快。

### 3. 检查依赖大小

```bash
# 分析依赖大小
npm install -g npm-check-updates
npx bundle-phobia [package-name]
```

### 4. 使用 Tree Shaking

确保 Vite 配置正确，未使用的代码会被自动移除。

### 5. 图片优化

- 使用 WebP 格式
- 压缩图片资源
- 使用 CDN 加载大图片

```bash
# Windows PowerShell
Get-ChildItem -Path "release" -Recurse -File |
  Sort-Object Length -Descending |
  Select-Object -First 10 Name, @{Name="Size(MB)";Expression={[math]::Round($_.Length/1MB, 2)}}
```

## 常见问题

### Q: 为什么 Electron 应用体积这么大？

A: Electron 应用包含完整的 Chromium 浏览器和 Node.js 运行时，这是正常的。通常体积在 100-200MB 之间。

### Q: 可以进一步减小体积吗？

A: 可以尝试：

- 使用 `electron-builder` 的 `compression: "maximum"`
- 移除不必要的依赖
- 使用更轻量的 UI 库（但你已经用了 Ant Design）
- 考虑使用 `electron-vite` 或其他优化工具

### Q: 900MB 是否正常？

A: **不正常**。正常应该在 120-200MB 之间。900MB 可能的原因：

- 同时打包了多个架构（x64 + ia32）
- 打包了不必要的文件（node_modules、源码等）
- 未启用压缩
- 包含了 source map
