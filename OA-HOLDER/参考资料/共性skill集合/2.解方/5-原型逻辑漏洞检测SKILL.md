# 原型逻辑漏洞检测SKILL

## 目标
- 识别并归并原型中的逻辑漏洞、流程断点与不一致
- 基于产品功能清单建立三向追踪（需求→原型→规则）
- 输出问题清单（含严重度、根因与复现路径）与修复建议
- 生成测试用例骨架，支撑后续验收

## 适用场景
- 原型评审或方案冻结前的系统化走查
- 需求变更后的快速一致性校验
- 验收前准备测试用例骨架与用例设计

## 输入
- 原型资产：HTML原型链接/文件夹、ASCII流程图、页面列表、交互说明
- 规则标注：角色权限矩阵、状态机定义、业务约束、异常与边界规则
- 功能清单：功能ID/模块/目标场景/角色权限/前置/关键操作/状态机/规则/异常边界/优先级/里程碑
- 变更记录：版本变更说明、changelog

## 输出
- 逻辑漏洞报告.md（问题清单+复现+根因+建议）
- 修复建议清单.md（按模块与优先级归档）
- 流程一致性图（SVG/ASCII，标注断点与异常分支）
- 需求-原型-规则追踪矩阵.xlsx（含映射状态与缺口）
- 覆盖率报告.md（功能/场景/边界维度）
- 测试用例骨架.xlsx（为验收SKILL生成详细用例的基础）

## 严重度分级
- S1：阻断主流程或导致不可逆数据错误
- S2：影响关键分支或高频场景
- S3：一致性/指引/文案类问题
- S4：优化建议与体验增强

## 工作流
1. 资产收集：解析HTML原型与ASCII流程，建立页面-交互-状态映射
2. 功能基线：以功能清单为锚，建立功能→页面/交互→规则的映射
3. 流程建模：构建用户旅程与状态机，标注角色/权限/数据流
4. 检测器执行：按检测器目录遍历场景并记录异常
5. 覆盖评估：计算功能/场景/边界覆盖率，标注未映射缺口
6. 问题归并：去重相似问题，确定严重度与影响范围
7. 修复建议：为每条问题生成可执行的修复路径与设计准则
8. 用例骨架：为缺陷与关键路径生成测试用例骨架，标注前置/数据/预期

## 检测器目录与要点
- 导航与返回路径：页面进出一致性、返回/关闭后状态保持；深链接与刷新上下文
- 会话与认证：登录态过期、跨端登录、权限变更即时生效与提示
- 权限与角色：越权操作、角色切换后菜单/动作/数据隔离
- 状态机：不可达/无终止/循环；跨状态操作合法性与按钮显隐
- 数据生命周期：创建-编辑-删除-恢复；草稿/撤销/重复提交防护
- 异常与空状态：请求失败/无数据/部分成功页态与引导
- 并发与重复提交：多次点击/网络抖动/竞态下的幂等与提示
- 时间与时序：定时任务、截止期、跨时区与夏令时
- 文案与术语一致性：同概念多称呼、按钮歧义、错误码与提示归一
- 国际化与可访问性：文案长度溢出、右到左、键盘可达性与焦点
- 多端一致性与响应式：PC/移动/小屏布局下关键操作可达性
- 价格与计费：税费/折扣/退款/多币种；试用到期与降配
- 通知与消息：触发条件、重复发送、关键信息遗漏、可关闭性
- 搜索与筛选组合：条件组合、清空重置、分页与排序一致性
- 文件上传与限制：大小/类型/并发、多文件中断恢复策略
- 表单校验：前后端一致、跨字段校验、错误定位与滚动引导
- 撤销与恢复：可逆操作范围、时间窗、数据一致性
- Feature Flag/AB：分组边界、开关回滚、跨流程一致性

## 覆盖指标
- 功能覆盖率 = 已映射功能数 / 功能总数
- 场景覆盖率 = 已建用例场景数 / 计划场景总数
- 边界覆盖率 = 已覆盖边界条件数 / 边界条件总数
- 未映射清单：列出未在原型或规则体现的功能/交互

## 报告结构模板（逻辑漏洞报告.md）
- 概述：版本、范围、方法、总览统计（按严重度）
- 问题清单：
  - ID / 模块 / 场景 / 严重度
  - 复现路径（步骤+期望+实际）
  - 影响范围（角色/数据/流程段）
  - 根因分析（设计/规则/交互/文案）
  - 修复建议（设计准则+落地步骤）
  - 关联用例（用例ID）
- 流程标注：一致性图与断点说明
- 建议合并策略：优先级与里程碑安排

## 测试用例骨架模板（测试用例骨架.xlsx）
- 用例ID
- 模块/页面
- 场景与目标
- 角色/权限
- 前置条件
- 步骤
- 预期结果
- 边界与异常
- 数据准备
- 严重度/优先级
- 关联缺陷ID

## 追踪矩阵（需求-原型-规则）
- 列：功能ID、模块、目标场景、优先级、状态
- 映射：页面/交互、规则/约束、用例ID、覆盖状态（已覆盖/部分/缺失）
- 缺口：未映射项自动标注与生成修复建议

## 与现有Skill的协同
- 原型设计SKILL：消费其HTML原型与ASCII流程产物进行检测
- 验收SKILL：将本Skill输出的用例骨架转换为可执行的详细用例与验收清单
- 产品规划SKILL：读取背景/用户/价值与核心场景，确保检测聚焦关键旅程

## 使用方式
- 触发关键词：逻辑漏洞检测 / 原型走查 / 一致性校验 / 用例骨架生成 / 追踪矩阵
- 操作指引：
  - 提供原型链接或文件夹、规则标注与功能清单
  - 指定角色范围与高优先级旅程
  - 选择输出（报告/追踪矩阵/覆盖率/用例骨架）
  - 运行检测并审阅输出，进入修复或验收阶段
