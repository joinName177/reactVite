# 原型评审SKILL

**目标与定位**
- 面向“原型评审”场景，统一多角色视角（高级产品经理、市场、研发/架构、设计/用户、数据/合规、运营/交付）进行结构化评审与结论输出
- 以“单文件HTML原型+规则标注”一致性为评审基线，形成变更清单、风险台账、优先级队列与可执行的验收门槛

**触发与输入**
- 触发时机：产品经理提交原型用于评审；进入方案对比后需形成“推荐结论与进入下一阶段依据”
- 必要输入
  - 原型：单文件HTML或可预览链接（含关键页面、交互说明、设计tokens）
  - 规则：RULE_DICT或原型内的data-rule-id/data-rule-detail映射，及数据/业务规则文档
  - PRD/范围：目标、边界、约束、成功指标、依赖清单
  - 对比材料：竞品/备选方案要点与打分表
  - 风险与假设：已识别风险、关键假设、备选缓解方案

**输出与交付物**
- 评审报告：多角色视角结论、维度评分、优先级决议、推荐方案与不采纳理由
- 变更清单：按严重度/优先级与模块归类，映射到具体原型DOM/规则ID，可直接用于更新
- 风险台账：技术/合规/市场/交付风险与缓解策略、触发条件、跟踪人
- 验收门槛：原型与规则一致性、关键交互与样式基线、可审计证据链
- 自动化候选：可由原型DOM与RULE_DICT半自动生成Gherkin用例与CSV清单

**评审维度与评分**
- 维度（0–5分，合规为硬门槛）：合规与制度映射、用户价值/场景、市场匹配/差异化、技术复杂度与可实现性、安全与隐私、数据质量与度量、业务可扩展性与版本化、运营与支持成本、可观测性与SLA、交付可行性与依赖
- 决策规则：合规不达标即否决；综合加权最高且满足关键路径/交付的为首选；分差接近时优先低风险与高交付可行性

**角色视角检查要点**
- 高级产品经理
  - 目标/边界与北极星指标一致，范围控制与里程碑划分
  - 原型-规则一致性与验收基线，版本化与生效/冻结管理
- 市场
  - 目标用户与价值主张清晰，竞品差异点、定价与推广路径
  - 增长杠杆、采纳阻力与替代成本分析
- 研发/架构
  - 关键路径实现性、接口与数据流、性能与容量、技术债与依赖
  - 安全与隐私、部署与回滚策略、可观测性与运维手册
- 设计/用户
  - 信息架构、关键任务流可达性、可用性与可访问性
  - 视觉与交互一致性（如Ant Design风格与tokens）
- 数据/合规
  - 规则映射与证据链、数据质量与留痕、制度与科目映射边界
- 运营/交付
  - 上线路径、培训与支持、问题处理SOP、监控告警与反馈闭环

**执行步骤**
- 准备：解析原型HTML与规则字典，建立页面-组件-规则的索引；收敛评审范围与关键用例
- 原型走查：按关键任务流进行可用性与一致性核对，记录差异并定位到DOM或规则ID
- 规则一致性：逐项比对RULE_DICT与原型展示/交互，标注冲突或缺失，形成修正建议与受影响范围
- 方案对比：统一维度打分，输出表格与结论；将“推荐结论与理由”置于对比表之后
- 风险与决议：归档风险台账，形成优先级队列（P0/P1/P2），明确负责人与截止
- 验收门槛：落地到具体用例与样式基线，生成自动化候选与定位器清单

**与其他技能的衔接**
- 原型设计：评审输出的变更清单与定位器用于更新原型与规则标注；界面风格遵循既定设计tokens
- 验收技能：本评审生成的验收门槛与用例/定位器直接作为验收清单输入，支撑审计证据链

**使用说明**
- 将原型HTML、规则字典与PRD投递到此skill，按模板输出报告、变更清单与风险台账
- 完成“方案对比→推荐结论→进入下一阶段”的决策链；未达合规门槛则回炉修改
