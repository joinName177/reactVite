# 功能使用手册编写器 - 使用说明

本 Skill 专为编写面向终端用户（特别是中老年客户群体）的操作手册而设计。它能将枯燥的需求与验收点，翻译成通俗易懂、带有温度的“傻瓜式”教程。

## 使用方式

1.  **准备工作**：
    *   *需求清单*：告诉 AI 功能是什么。
    *   *验收清单*：告诉 AI 具体操作步骤和预期结果。
2.  **触发 Skill**：呼叫“功能使用手册编写器”。
3.  **提供输入**：同时提供上述两份资料（文本或路径）。
4.  **接收产出**：AI 将输出一份结构完整的 Markdown 手册，你可以直接复制到 Word 中进行排版微调。

## 核心价值与约束

-   **特定读者画像**：默认针对 **45-55 岁** 客户群体。这意味着：
    -   字体与格式要清晰（AI 会通过加粗重点来实现）。
    -   完全摒弃技术术语（如“接口”、“回调”），全部转换为生活化语言。
-   **步骤化教学**：将每一个操作都拆解为 `1. 2. 3.` 的步骤，不跳跃，不假设用户“应该懂”。
-   **Word 友好**：输出格式针对 Word 文档进行了优化，方便后续处理。

## 案例与示例

### 场景示例
系统上线了“自动对账”功能，研发说“通过 CronJob 每天凌晨拉取支付网关账单进行对账”，现在需要写手册告诉财务大姐怎么用。

**用户输入**：
> 需求：自动对账系统，每天凌晨 2 点运行。
> 验收点：管理员登录后台，点击“财务中心 -> 对账记录”，查看当天的对账结果（成功/失败）。如果失败，点击“重新对账”按钮。

**AI 响应结果（片段）**：

#### 财务中心

**1. 查看自动对账结果**
- **这是什么**：系统每天半夜会自动帮您核对前一天的账目，您早上来只需要看一眼结果对不对。
- **如何操作**：
  1. 登录系统，找到左侧菜单栏的 **“财务中心”**。
  2. 点击 **“对账记录”** 按钮。
  3. 在列表中查看今天的记录：
     - 如果显示 **“成功”**（绿色），说明账目没问题。
     - 如果显示 **“失败”**（红色），请点击后面的 **“重新对账”** 按钮试一次。
- **温馨提示**：系统通常在凌晨 2 点工作，建议您每天早上 9 点以后再查看，确保数据都跑完了。
