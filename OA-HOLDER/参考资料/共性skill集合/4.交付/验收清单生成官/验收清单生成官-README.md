# 验收清单生成官 - 使用说明

本 Skill 旨在解决交付验收阶段“需求转验收项”耗时且易漏的问题，帮助项目经理或测试人员快速生成结构化、可执行的功能验收 Checklist。

## 使用方式

1.  **准备工作**：整理好你的需求清单（通常是 Excel、Markdown 或即时通讯软件中的文本）。
    *   *必备信息*：功能点名称、功能描述。
    *   *推荐信息*：影响范围、所属模块。
2.  **触发 Skill**：呼叫“验收清单生成官”。
3.  **提供输入**：直接粘贴需求清单内容，或提供包含需求的文件路径。
4.  **接收产出**：AI 将输出按功能分组的 Markdown Checklist，你可以直接复制到验收文档或任务管理工具中。

## 核心价值与约束

-   **聚焦功能验证**：本 Skill 刻意忽略了非功能性需求（如性能指标）和代码实现细节，专注于“用户可见的功能”。
-   **去繁就简**：输出的 Checklist 不包含测试等级（P0/P1）、责任人等管理字段，只保留“测什么”和“预期结果”。
-   **结构化输出**：自动按功能模块分组，避免清单杂乱无章。

## 案例与示例

### 场景示例
项目经理刚收到一份《V1.2 订单导出功能》的需求文档，需要在 10 分钟内整理出一份给客户确认的验收清单。

**用户输入**：
> 请根据以下需求生成验收清单：
> 1. 订单列表页新增“导出”按钮，仅管理员可见。
> 2. 点击导出后，弹出时间选择器，默认选择最近 7 天。
> 3. 导出格式为 Excel，包含订单号、金额、下单人。
> 4. 导出期间显示 loading 状态，不可重复点击。

**AI 响应结果（片段）**：

### 功能分组：订单导出
- [ ] **验收项：导出按钮权限**
    - **验证点**：使用管理员账号登录，查看订单列表页右上角。
    - **预期结果**：可见“导出”按钮；切换普通员工账号登录，按钮不可见。
- [ ] **验收项：时间选择器默认值**
    - **验证点**：点击“导出”按钮，检查弹出的时间选择器。
    - **预期结果**：默认时间范围为最近 7 天（如今日 10号，则范围为 3号-10号）。
- [ ] **验收项：重复点击防护**
    - **验证点**：在导出 Loading 期间再次点击“导出”按钮。
    - **预期结果**：按钮置灰或无响应，系统不发起新的导出请求。
