# 我的编排 Skill 规则 — 使用说明

本 README 说明如何使用「我的编排 Skill 规则」创建**多 Skill 编排型 Skill**，以及本次创建时的决策记录与示例。

---

## 一、使用方式

### 何时会用到本规则

- 你要**新建一个编排型 Skill**：按固定顺序串联多个已有 Skill，形成从用户输入到多步产出的流水线。
- 你希望**先和 AI 把编排想法聊清楚**，再让 AI 产出 SKILL.md 与 README，即采用「先提问、再产出」的创建流程（与「我的 Skill 创建规则」一致）。
- 你希望编排的**执行节奏统一、可预期**：每步先提问、等用户答复、判断是否还要问、没有要问的再询问是否生成文档、用户确认后才输出、然后才进入下一步。
- 你手头已有某个编排 Skill（如「客户想法到营销方案编排」），希望把其中的**流程与关键提炼成通用规则**，用于以后创建更多编排 Skill。

### 创建编排 Skill 时的流程：先提问、再产出

当你按本规则**创建**一个新的编排 Skill 时，AI **必须**按以下节奏执行，不得跳过：

1. **第一轮：AI 必须先提问，不写稿**  
   你说明需求后，AI 的**第一轮回复必须是「向你提问」**，用于了解你的所有编排诉求（编排哪几个 Skill、顺序、谁用、场景、每步输入输出、是否要「生成文档前确认」等）。AI **禁止**在第一轮就输出编排 Skill 的 SKILL.md 或 README 全文。

2. **多轮提问直到 AI 没有问题**  
   AI 会根据你的描述主动、连续地提问（可多轮）。**你需要逐条答复**。每轮你答复后，AI 会判断是否还有要问的：若还有，只再输出下一批问题；若没有了，会问你「是否要生成本次的编排 Skill 文档？您确认后我再输出。」

3. **你确认后 AI 才产出**  
   **只有**在你明确表示「可以」「生成吧」「要」或「可以出稿了」之后，AI 才产出编排 Skill 的 SKILL.md 与配套 README。

**要点**：AI 会**根据你的需求先向你提问**，**了解清楚你所有的编排诉求之后**，再在某一方向生成编排 Skill。未完成多轮提问、未得到你「可以出稿了」的确认前，AI 不应输出可保存的 SKILL.md 全文（可输出提纲或片段供你确认）。

### 推荐用法

1. **创建新编排 Skill 时**  
   @ 或拖入「我的编排 Skill 规则」，说明你要做的编排（例如：先想法拆解、再营销方案）。AI 会先按「创建编排 Skill 的流程」向你提问，你逐条答复，直到 AI 没有更多问题且你确认可以出稿，再一次性产出 SKILL.md 与 README。

   产出的 SKILL.md 将按规则中的结构组织：何时使用、编排对象与顺序、执行流程（每步：输入、执行、提问与澄清、生成文档前确认、输出、交接）、关联 Skill 的提问与执行节奏、编排规则、简要自检、YAML 与 README 建议。

3. **重点遵守**  
   - **提问节奏**：先问 → 用户答 → 仅判断是否还要问 → 若还要问则只继续提问（不输出文档、不进入下一步）→ 直到没有要问的 → 询问「是否要生成本次文档」→ 用户确认后再输出 → 再进入下一步。  
   - **禁止两种跳过**：还有问题要问时就输出文档或进入下一步；未征得用户「是否生成」的确认就输出文档。

4. **配套 README**  
   按「我的 Skill 创建规则」，为编排 Skill 编写 README，包含：使用方式、本次创建中的确认与否定、使用案例、结果示例。

### 和 AI 的配合要点

- 若让 AI 帮你**起草**新的编排 Skill，可说明「按我的编排 Skill 规则」来写，并指明要编排哪几个 Skill、顺序与路径。
- 若让 AI **检查**已有编排 Skill，可要求对照「我的编排 Skill 规则」做自检，看是否缺节、是否写清提问节奏与禁止跳过。

## 二、本次创建中的确认与否定

以下来自从「市场类-客户想法到营销方案编排」中提炼流程与关键、并泛化为「我的编排 Skill 规则」时的约定。

### 确认采用的方案 / 观点

- **来源**：从「市场类-客户想法到营销方案编排」的流程与关键点抽象出通用规则，用于创建任意「多 Skill 编排」型 Skill。  
- **文档结构**：编排 Skill 必须包含「何时使用」「编排对象与顺序」「执行流程」「关联 Skill 的提问与执行节奏」「编排规则」「简要自检」；执行流程中每步必须包含输入、执行、提问与澄清、生成文档前确认、输出、交接。  
- **提问与输出节奏**：每次用户答复后只做「判断是否还要问」；若还要问则只继续提问、不输出文档、不进入下一步；当没有要问的时，必须先询问「是否要生成本次文档」，用户明确确认后才输出该步文档；当前步完全结束后再进入下一步。  
- **禁止两种跳过**：明确写入编排规则——（1）还有问题要问时不得输出文档或进入下一步；（2）未询问且未得到用户确认前不得输出文档。  
- **产出物**：各步独立产出，不强制要求再写汇总文档。  
- **路径引用**：执行各步前先读取对应 Skill 文件，再按其指令执行；路径在「编排对象与顺序」表格中给出。  
- **创建方式**：本规则本身按「我的 Skill 创建规则」产出，即 README 含使用方式、确认与否定、使用案例、结果示例。  
- **创建编排 Skill 的流程**：使用本规则创建新的编排 Skill 时，AI 必须先提问（阶段一、二），直到没有更多问题且用户确认「可以出稿了」，再产出 SKILL.md 与 README（阶段三）；与「我的 Skill 创建规则」的「先 ASK 澄清、再产出」保持一致。

### 否定的方案 / 不采用的写法

- **不**在编排文档中省略「生成文档前确认」环节；必须明确「先询问是否生成、用户确认后再输出」。  
- **不**允许在还有待追问问题时输出当前步文档或进入下一步。  
- **不**在规则中写死某一业务域（如仅校园餐）；规则泛化为适用于任意多 Skill 编排。

### 其他约定

- 「编排型 Skill」指：按固定顺序调用多个已有 Skill，每步有明确输入/输出，上一步产出作为下一步输入。  
- 被编排的 Skill 本身保持独立，编排文档只规定顺序、输入输出与节奏，不改写被编排 Skill 的内部逻辑。

## 三、使用案例

### 案例 1：新建「需求拆解 → 技术方案」编排 Skill

- **情境**：希望做一个编排，先执行「需求一句话拆解」Skill，再执行「技术方案编写」Skill。  
- **用法**：@「我的编排 Skill 规则」，说明要编排的两个 Skill 名称与路径、第一步输入来自用户、第二步输入来自第一步的拆解文档。  
- **预期**：按规则写出编排 SKILL.md，含编排对象与顺序表、两步的执行流程（每步含生成文档前确认）、提问与执行节奏、编排规则、自检；并写好 README。

### 案例 2：检查已有编排 Skill 是否合规

- **情境**：已有「客户想法到营销方案编排」，想确认是否缺少「生成文档前确认」或「禁止两种跳过」等关键点。  
- **用法**：@「我的编排 Skill 规则」，请 AI 对照规则检查该编排 Skill 的 SKILL.md。  
- **预期**：得到逐节对照结果，并标出缺失或建议补充的句子（如「是否要生成本次的 XXX 文档？您确认后我再输出」）。

### 案例 3：从零设计一个三步编排

- **情境**：要设计「客户需求 → 想法拆解 → 产品方案 → 开发任务拆分」三步编排。  
- **用法**：按「我的编排 Skill 规则」先列出编排对象与顺序表，再为每一步写执行流程（输入、执行、提问与澄清、生成文档前确认、输出、交接），最后写提问与执行节奏、编排规则、自检。  
- **预期**：得到一份结构完整、节奏统一的三步编排 Skill 文档，可直接或稍作修改后保存为 SKILL.md。

## 四、结果示例

### 编排 Skill 文档结构摘要

按本规则创建的一份编排 Skill，其 SKILL.md 大致结构如下：

```markdown
# XXX 编排 Skill

## 何时使用
（谁、什么场景、如何触发）

## 编排对象与顺序（固定）
（表格：顺序、被编排 Skill、路径）

## 执行流程
### 第一步：执行「A」
（输入、执行、提问与澄清、生成文档前确认、输出、交接）
### 第二步：执行「B」
（同上）

## 关联 Skill 的提问与执行节奏（必须遵守）
（多轮提问节奏、生成文档前必须经用户确认、当前 Skill 完全结束后再进入下一个；总结句）

## 编排规则（必须遵守）
（严格顺序、提问节奏、禁止两种跳过、产出物不汇总、输入来源、路径引用）

## 简要自检（每步结束后）
（第 K 步结束：已输出完整 XXX 文档…）
```

### 关键句示例（可写入编排文档）

- **生成文档前确认**：「我这边没有更多问题了。是否要生成本次的 [本步文档名称] 文档？您确认后我再输出。」  
- **禁止跳过**：「不得在还有要追问的问题时输出当前步文档或进入下一步；不得在未询问且未得到用户确认生成前输出该步文档。」  
- **节奏总结**：「先问 → 用户答 → 仅判断是否还要问 → 若还要问则只输出下一批问题（不输出文档、不进入下一步）→ 重复直到没有要问的 → 问用户是否要生成本次文档 → 用户确认后再输出文档 → 然后才进入下一个关联 Skill。」
