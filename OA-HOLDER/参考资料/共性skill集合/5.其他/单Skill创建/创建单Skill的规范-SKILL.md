# 我的 Skill 创建规则

本规则定义「按我的方式」创建单个新 Skill 的完整流程：**先通过对话摸清所有想法，再一次性产出 Skill 文档与配套 README**。仅针对当前要创建的那一个 Skill，不依赖、不引用已有 Skill。

---

## 何时使用

- 用户说要用「我的方式」或「按你的（指用户）规则」创建 skill 时。
- 用户说「我想创建一个 skill，先帮我理清想法再写」或类似表述时。
- 用户明确要求：先 ASK 模式聊想法、再出文档时。

**不触发**：用户只是问「怎么创建 skill」「skill 怎么写」等一般性问题，且未说明要用「先澄清再产出」的流程时。

---

## 流程总览

1. **阶段一：仅讨论（ASK）**  
   用户在 ASK 模式下说明：想要一个什么样的 skill、解决什么问题、给谁用。AI 只进行分析与提问，不生成最终 skill 文件。

2. **阶段二：反复提问直到摸清**  
   AI 根据当前描述进行分析，并主动、连续地提问（可多轮）。每轮可包含：对目标/范围/场景的确认、对可选方案的取舍、对边界与约束的澄清。直到**当前会话内**已摸清用户的所有想法（用户确认或等价表述）。

3. **阶段三：产出交付物**  
   仅在用户确认「想法已摸清」「可以出稿了」或等价意思之后，再产出：
   - **Skill 文档**：即最终的 `SKILL.md`（含 YAML frontmatter 与正文）。
   - **配套 README**：见下方「README 必须包含的内容」。

---

## 阶段一：仅讨论（ASK）

- 默认假定用户处于 **ASK 模式**或「只讨论不直接改库」的意图。
- **范围限定**：只讨论「当前要创建的这一个 skill」，不引用、不依赖项目里已有其他 skill 的内容与结构。
- AI 先复述或概括用户的想法，再说明将通过多轮提问把需求摸清，然后进入阶段二。

---

## 阶段二：反复提问直到摸清

- **分析**：根据用户当前描述，提炼出：目标、使用场景、谁用、输入/输出、与其它流程的边界等。
- **提问方式**：  
  - 一次可问多个相关问题，但主题清晰。  
  - 可给出 2～3 个可选方案让用户选「要 / 不要 / 再改」。  
  - 对模糊处（例如「有时」「看情况」）主动追问成可执行规则。
- **结束条件**：  
  - 用户明确表示「就这些」「没了」「可以写了」等；或  
  - 用户对关键问题（目标、范围、产出物、约束）都已做出选择或确认。  
- **记录**：在会话中自然记录用户的**确认与否定**（例如：采用 A 否掉 B、不做 C）。这些将用于 README 的「本次创建中的确认与否定」部分。

---

## 阶段三：产出交付物

仅在阶段二结束条件满足后执行。

### 1. Skill 文档（SKILL.md）

- 符合 Cursor Skill 规范：`name`、`description` 及正文。
- 内容完全基于**当前会话中已确认**的目标、范围、步骤与约束编写；不引入未在会话中确认的假设。
- 若用户指定了存放路径或命名，按用户要求；未指定则给出建议路径（如 `.cursor/skills/<skill-name>/SKILL.md`）。

### 2. 配套 README（与 SKILL 同目录或用户指定位置）

README 必须包含以下四类内容：

| 部分 | 说明 |
|------|------|
| **使用方式** | 本 skill 的触发场景、推荐用法（例如：在 ASK 模式下先说想法），以及如何与 AI 配合完成一次「按我的方式创建 skill」的流程。 |
| **本次创建中的确认与否定** | 本次创建过程中，用户确认采用的方案、否掉的方案、以及重要观点或约束（可从会话总结，用列表或简短段落）。 |
| **使用案例** | 1～3 个简短案例：用户在什么情境下会用到本 skill、典型输入（用户说什么/提供什么）、期望的交互过程。 |
| **结果示例** | 可选的产出示例：例如某次创建后得到的 Skill 文档结构摘要、README 片段等，用于说明「按此流程会得到什么样的结果」。 |

若用户希望 README 更简短，可合并「使用案例」与「结果示例」为「案例与示例」。

---

## 会话中的注意事项

- **不提前写稿**：在用户未确认「想法已摸清」前，不输出最终的、可保存的 SKILL.md 全文；可输出提纲或片段用于确认。
- **术语一致**：一旦用户选定了某说法（如「ASK 模式」「只讨论」），后续提问与 README 中保持一致。
- **一次只做一个 skill**：同一会话内只完成「当前这一个」skill 的创建；若用户想再做一个，视为新会话、新一次「按我的方式创建」。

---

## 简要自检清单（产出前）

- [ ] 用户已确认或等价表示「想法已摸清」。
- [ ] SKILL.md 的 name、description 与正文与当前会话结论一致。
- [ ] 配套 README 已包含：使用方式、确认与否定、使用案例、结果示例（或合并后的案例与示例）。
