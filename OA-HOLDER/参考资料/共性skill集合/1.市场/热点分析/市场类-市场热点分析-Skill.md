# 🔥 热点分析 Skill

## ℹ️ 基本信息
- **核心用户**: 项目经理 (PM)、市场/战略、销售主管
- **解决问题**: 没时间盯行业动态、不知道某方向某段时间有什么热点、热点与业务/机会的关系不清晰。
- **输入**: **一个**方向（如团餐数字化、校园食品安全）、**一段时间**（由用户描述，如「近一周」「3 月 1 日到 3 月 10 日」）。
- **输出**: 各平台热点表格（微信、今日头条、抖音、百度、政府政策网、微博、知乎、快手、小红书等各搜一遍后整合，**链接列须为可点击的超链网址**）+ 市场情况与预测 + 应对方案（以资深行业市场销售专家身份，按「情况→预测→方案」撰写）。

## 📌 用法与提示词

**何时使用**：用户明确要看**某一方向**、**某一段时间内**的市场热点，并希望得到整合后的热点表与市场分析、预测及应对方案时，调用本 Skill。

**推荐提示词示例**：
- 「帮我看一下 [方向] 最近 [时间] 的市场热点，用一下热点分析。」
- 「用热点分析 Skill，查 [方向] [时间] 的热点，并分析市场和给方案。」
- 「[方向]，[时间] 内的热点搜一下，微信头条抖音百度等都看，然后出热点表和分析。」

**约定**：本 Skill 只处理**一个方向**；时间范围完全由用户当次描述决定，不写死默认。

---

## ⚠️ 两项硬性约束（必须遵守）

1. **时间范围必须准确**
   - 用户给出的时间描述（如「最近3天」「近一周」「3 月 1 日到 3 月 10 日」）即**唯一有效时间窗口**。
   - 执行时先根据**当前日期**与用户描述推算出**具体起止日期**（例如「最近3天」= 今天往前推 3 天的起止日），并在报告标题下**明确写出**「时间窗口：YYYY年MM月DD日～YYYY年MM月DD日」。
   - **热点表中只纳入发布时间/通报时间/报道时间落在该时间窗口内的内容**。若某条内容的日期早于或晚于该窗口（如用户要「最近3天」却出现 2024 年、2025 年初等），**一律不得列入热点表**；无明确日期且无法从正文推断在窗口内的，也**不得纳入**。
   - 不得用「近期」「近日」等模糊表述替代具体日期；不得为凑条数而放宽时间范围。

2. **内容必须与用户关注方向直接相关，不发散**
   - 检索与呈现的热点必须**紧扣用户当次给出的方向**（如「校园食品安全」），仅纳入与该方向**直接相关**的新闻、政策、事件（如：校园/学校食堂/学生餐/校园周边食安等）。
   - **不扩大、不联想**：不纳入仅泛泛涉及「食品安全」「餐饮监管」而与校园/学生餐无直接关系的热点；不纳入弱相关或需要联想才能扯上关系的内容。
   - 分析部分**仅基于**上述筛选后的热点撰写，不引入表外信息，不发散推理。

## 🤖 执行流程（必须按顺序）

### Step 1：生成搜索查询并执行多平台搜索

1. **调用脚本**：运行本 Skill 同目录下 `scripts/fetch_hotspots.py`，传入用户给的**方向**和**时间描述**。
   - 示例：`python scripts/fetch_hotspots.py --direction "团餐数字化" --time "近一周"`
   - 脚本会按**优先级顺序**输出各平台建议的搜索关键词列表（见下方平台与优先级）。
2. **按平台分别搜索**：根据脚本输出的关键词，对以下**各入口按优先级依次执行搜索**（使用 Web Search 或浏览器）：
   - **优先级 1（最高）**：**微信**、**今日头条**、**抖音**、**百度** — 必须覆盖；
   - **优先级 2**：**政府政策公布网**（如政府网、各部委官网、地方政务平台的政策/通报）；
   - **优先级 3**：**微博**、**知乎**、**快手**、**小红书** — 视时间与结果量选择性覆盖。
   - 各平台具体搜法见脚本输出的关键词；政府类可搜「[方向] 政策 政府」「[方向] 通报 监管 近期」等。
3. **只使用真实搜到的内容**：从各次搜索结果中提取**标题、真实 URL、摘要、日期**等；不编造未出现的新闻或数据。**每条热点必须保留可访问的完整网址**，供 Step 2 填超链。
4. **先算时间窗口再筛选**：根据用户时间描述与当前日期算出**具体起止日**；检索到的每条内容必须**有明确日期且落在该窗口内**才可进入热点表，否则排除。仅呈现与用户给出的**方向直接相关**的热点，不发散。

### Step 2：整理成热点表

- **时间与相关性筛选**：只将**日期落在用户要求的时间窗口内**且**与用户给出的方向直接相关**的条目放入下表；超出窗口或仅弱相关的一律不列入。报告开头注明「时间窗口：YYYY年MM月DD日～YYYY年MM月DD日」。
- 将筛选后的内容整合为一张表，表格列如下（有则填，无则空）：

| 列名 | 说明 |
|------|------|
| 序号 | 1, 2, 3… |
| 标题/主题 | 热点标题或主题 |
| 来源平台 | 微信 / 今日头条 / 抖音 / 百度 / 政府政策网 / 微博 / 知乎 / 快手 / 小红书 等 |
| 日期 | 若搜索结果中有则填 |
| 链接 | **必填**：从搜索结果中提取的**完整 URL**，并以 **Markdown 超链**形式书写，即 `[原文](完整网址)` 或 `[标题](完整网址)`，确保读者可点击跳转；不可用「多源转载」等文字替代。无链接时填「—」。 |
| 一句话摘要 | 核心信息摘要 |
| 类型 | 政策 / 融资 / 产品发布 / 热点事件 / 其他 |
| 热度/阅读量 | 若搜索结果中有则填，否则留空 |

### Step 3：市场分析与应对（专家人设）

以**资深行业市场销售专家**的身份，基于**且仅基于**上表热点，撰写分析。结构不固定，但每一块都要把以下三者写在一起：
- **某个情况是什么**：从热点中提炼出的市场现象、政策/资本/需求变化等；
- **市场预测是什么**：基于该情况对短期趋势的判断；
- **应对方案是什么**：针对该情况与预测，给出的 1～3 条可执行动作（市场/销售/产品等均可）。

可写多块（例如：政策与监管一块、资本与竞争一块、需求与痛点一块），每块内部都遵循「情况 → 预测 → 方案」。**仅基于热点表中的条目撰写，不编造、不引入表外信息、不发散。**
