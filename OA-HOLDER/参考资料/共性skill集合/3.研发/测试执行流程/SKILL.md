# 后端测试执行流程

定义后端测试执行的标准流程。

## 何时使用

- 组织测试执行
- 规划测试活动
- 规范测试流程

## 执行阶段

### 阶段1: 测试准备

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 确认测试范围 | 测试负责人 | 测试范围文档 |
| 准备测试环境 | 运维/开发 | 可用环境 |
| 准备测试数据 | 测试/开发 | 测试数据集 |
| 评审测试用例 | 测试团队 | 评审通过用例 |
| 准备自动化脚本 | 自动化测试 | 可执行脚本 |

### 阶段2: 测试执行

| 任务 | 方法 | 工具 |
|------|------|------|
| 接口自动化测试 | pytest执行 | requests, pytest |
| 功能测试 | 手动验证 | 接口文档, Postman |
| 性能测试 | 压力测试 | locust, jmeter |
| 安全测试 | 漏洞扫描 | 安全扫描工具 |

### 阶段3: 问题跟踪

```
发现问题
├── 记录Bug（标题、复现步骤、期望结果）
├── 提交到缺陷管理系统
├── 开发人员修复
├── 测试人员验证
└── 关闭或重新打开
```

### 阶段4: 测试报告

| 内容 | 说明 |
|------|------|
| 测试概况 | 用例数、通过率、执行时间 |
| Bug统计 | 新增、修复、遗留 |
| 风险评估 | 上线风险评估 |
| 建议 | 是否可上线 |

### 阶段5: 测试复盘

- 测试过程中的问题
- 用例覆盖度分析
- 改进建议
- 经验沉淀

## 执行检查清单

- [ ] 测试环境可用
- [ ] 测试数据准备完成
- [ ] 用例评审通过
- [ ] 自动化脚本就绪
- [ ] 缺陷跟踪系统可用
- [ ] 报告模板准备

## 何时使用

用户提到「测试流程」「测试组织」「测试管理」或需要规范测试执行时，自动应用本技能。
