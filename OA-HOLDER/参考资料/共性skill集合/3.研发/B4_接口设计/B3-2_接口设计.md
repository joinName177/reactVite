### 🔵 B3-2 接口设计
> **归属板块**：产品/研发侧 | **主导角色**：后端, 前端, 架构师 | **工具**：Trae/Cursor + `接口设计Skill`
> *⏳ 依赖 B3-1 方案预设计产出；输出供 B4-1 研发方案总纲使用*

*   **🎯 关键动作**：基于需求设计结果，从多个维度对接口进行系统性设计，确保接口语义清晰、结构规范、安全可靠、易于维护。
*   **🤖 Skill**：按需调用下方 Skill 矩阵中的专家，完成从接口结构到安全、版本、文档的全链路设计。
*   **📥 输入**：需求解析报告 + 方案架构文档 + 数据模型 + 接口草案（B3-1 产出）
*   **📤 输出**：完整接口定义文档 + OpenAPI/Swagger YAML + 错误码规范 + 接口安全方案 + Mock 数据（若有合规/网络安全要求：安全合规审计清单 + 网络安全基线说明）
*   **✅ 验证闭环**：**契约闭环**（接口覆盖所有业务场景；前后端对齐无歧义；安全合规通过）

### 🧩 细分领域 Skill 矩阵

| 序号 | Skill 名称 | 中文名称 | 核心用户 | 解决的核心问题 | 核心功能/预期产出 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 1 | **RESTful API Architect** | RESTful接口架构师 | 后端/架构师 | 如何设计语义清晰、符合 REST 规范的接口路径和方法？ | **功能**：设计资源路径、HTTP 方法、状态码映射。<br>**产出**：接口清单、路径规范说明、HTTP 方法使用指南。 |
| 2 | **Request/Response Designer** | 请求响应结构设计师 | 前端/后端 | Request Body 和 Response Body 的字段如何精确定义？ | **功能**：定义每个接口的入参/出参结构、字段类型、必填项、枚举值。<br>**产出**：JSON Schema、TypeScript Interface、字段说明表。 |
| 3 | **Error Code Standardizer** | 错误码规范师 | 全体研发 | 接口异常如何统一表达？错误码如何分层管理？ | **功能**：设计全局错误码体系，区分系统级/业务级/参数级错误。<br>**产出**：错误码清单、统一响应格式规范、前端错误处理指南。 |
| 4 | **API Security Designer** | 接口安全设计师 | 后端/安全 | 接口如何防止越权、重放攻击、参数注入等安全风险？ | **功能**：设计认证方案（JWT/OAuth）、权限校验、参数过滤、限流策略。<br>**产出**：接口安全方案文档、鉴权流程图、风险点清单。 |
| 5 | **Pagination & Filter Designer** | 分页与过滤设计师 | 前端/后端 | 列表接口如何设计分页、排序、过滤，兼顾性能与灵活性？ | **功能**：设计分页参数规范、过滤条件结构、排序字段定义。<br>**产出**：列表接口通用参数规范、游标/偏移分页方案对比、示例。 |
| 6 | **OpenAPI Doc Generator** | OpenAPI文档生成师 | 全体研发/测试 | 如何将接口设计快速转化为可交付的标准 API 文档？ | **功能**：将接口定义转换为 OpenAPI 3.0 YAML/JSON 格式。<br>**产出**：OpenAPI YAML 文件、Swagger UI 可渲染文档、接口变更日志。 |
| 7 | **Mock Data Factory** | Mock数据工厂 | 前端/测试 | 后端接口未就绪时，前端和测试如何获得真实可用的 Mock 数据？ | **功能**：根据接口定义生成符合业务语义的 Mock 数据和 Mock Server 配置。<br>**产出**：Mock JSON 数据集、MSW/Apifox Mock 配置、边界值 Mock 样例。 |
| 8 | **API Version Strategist** | 接口版本策略师 | 架构师/后端 | 接口迭代时如何管理版本，避免破坏性变更影响已有调用方？ | **功能**：设计接口版本策略（URL版本/Header版本），定义兼容性规则。<br>**产出**：版本管理规范、Breaking Change 检测清单、废弃接口迁移方案。 |
| 9 | **Security Compliance Auditor** | 安全合规审计 | 后端/安全 | 接口与数据是否满足等保、行业合规与审计可追溯要求？ | **功能**：对照合规标准做接口与数据侧检查，输出审计日志要求、敏感数据分类与留存策略。<br>**产出**：合规项清单、审计日志规范、敏感数据与脱敏策略。 |
| 10 | **Network Security Advisor** | 网络安全顾问 | 架构师/运维/安全 | 传输层与部署层有哪些安全要求？HTTPS、安全头、网络防护如何约定？ | **功能**：定义 HTTPS/TLS、安全头（CORS/CSP/HSTS）、与部署/网络相关的安全基线。<br>**产出**：网络安全基线说明、与接口安全方案的衔接说明。 |

### 🛤️ Skill 组合路径建议

#### 🟢 场景一：全新模块接口设计（从 0 到 1）
*   **解决问题**：基于需求方案，完整设计一套新模块的接口体系。
*   **建议组合**：`RESTful接口架构师` -> `请求响应结构设计师` -> `错误码规范师` -> `接口安全设计师` -> `Mock数据工厂` -> `OpenAPI文档生成师`（若有等保或合规要求，可增加 `安全合规审计`、`网络安全顾问`）

#### 🔵 场景二：列表查询类接口设计**
*   **解决问题**：涉及复杂筛选、分页、排序的列表接口如何设计？
*   **建议组合**：`RESTful接口架构师` -> `分页与过滤设计师` -> `请求响应结构设计师` -> `Mock数据工厂` -> `OpenAPI文档生成师`

#### 🟣 场景三：存量接口迭代升级
*   **解决问题**：已有接口需要新增字段或调整结构，如何保证向后兼容？
*   **建议组合**：`接口版本策略师` -> `请求响应结构设计师` -> `错误码规范师` -> `OpenAPI文档生成师`

#### 🟠 场景四：接口安全加固
*   **解决问题**：现有接口存在安全隐患，如何系统性加固？
*   **建议组合**：`接口安全设计师` -> `安全合规审计` -> `网络安全顾问` -> `错误码规范师` -> `OpenAPI文档生成师`
