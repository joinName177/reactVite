# 测试报告

编写阶段或版本测试报告。执行前先阅读项目 README.md 与测试计划、执行结果等已有产出。

## 原则

- 数据来源于测试计划、执行结果与缺陷统计，不捏造。
- 结论明确：是否达到发布标准、遗留风险与建议。
- 所有产出物保存到项目根目录下的 **测试** 文件夹。

## 输入

- 测试计划
- 测试执行结果（通过/失败/阻塞/跳过统计）
- 缺陷统计（可选）：按严重程度、模块、状态等
- 版本/迭代信息与发布标准（如有）

## 输出

保存到 `测试/` 文件夹，建议文件名如 `测试报告-<版本>-<日期>.md`：

1. **报告摘要**：版本、测试周期、结论（通过/有条件通过/不通过）与一句话说明。
2. **测试执行概况**：用例总数、执行数、通过率、失败/阻塞数。
3. **缺陷概况**：总数、按严重程度分布、按模块分布、遗留未修复数（如有）。
4. **测试覆盖**：功能/模块覆盖说明或覆盖矩阵摘要。
5. **遗留风险与建议**：未修复缺陷影响、未覆盖范围、发布建议与后续改进。

## 报告模板

```markdown
# 测试报告 - [版本] - [日期]

## 1. 报告摘要
- **版本**：...
- **测试周期**：...
- **测试结论**：通过 / 有条件通过 / 不通过
- **摘要说明**：...

## 2. 测试执行概况
| 项目 | 数量 |
|------|------|
| 用例总数 | ... |
| 已执行 | ... |
| 通过 | ... |
| 失败 | ... |
| 通过率 | ...% |

## 3. 缺陷概况
| 严重程度 | 数量 | 已修复 | 遗留 |
|----------|------|--------|------|
| 致命/严重/一般/轻微 | ... | ... | ... |

## 4. 测试覆盖
- 覆盖模块/功能简述或覆盖矩阵摘要

## 5. 遗留风险与建议
- 遗留问题及影响
- 发布建议
- 后续改进建议
```

## 流程

1. **汇总数据**：从执行结果与缺陷列表提取关键数字。
2. **计算通过率与分布**：便于结论有据可依。
3. **给出结论**：对照发布标准给出是否建议发布。
4. **写清风险与建议**：便于项目决策与后续迭代。

## 何时使用

用户提到「测试报告」「测试总结」「质量报告」「版本汇报」或要求根据执行结果编写报告时，自动应用本技能。
