### 🟡 B3-1 方案预设计
> **归属板块**：产品/研发侧 | **主导角色**：产品经理, 架构师 | **工具**：Trae/Cursor + `需求理解Skill`
> *⏳ 依赖 B1 需求契约化产出；输出供 B4-1 研发方案总纲使用*

*   **🎯 关键动作**：深度解析需求意图，设计研发方案架构，生成配套文档与流程图，定义数据模型。
*   **🤖 Skill**：按需调用下方 Skill 矩阵中的能力，完成从需求理解到方案落地的全链路设计。
*   **📥 输入**：需求文档 / PRD / 用户故事
*   **📤 输出**：需求解析报告 + 方案架构文档 + 流程图 + 数据模型 + 接口草稿（可选：需求与现有系统关联检测报告、性能与扩展性说明）
*   **✅ 验证闭环**：**逻辑闭环**（需求无歧义；方案可落地；模型覆盖业务场景）

### 🧩 细分领域 Skill 矩阵

| 序号 | Skill 名称 | 中文名称 | 核心用户 | 解决的核心问题 | 核心功能/预期产出 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 1 | **Requirement Analyst** | 需求解析 | 产品经理/研发 | 需求描述模糊，如何提炼核心意图、边界条件和验收标准？ | **功能**：拆解需求为用户故事、场景、边界条件。<br>**产出**：需求解析报告、澄清问题清单、验收标准。 |
| 2 | **Solution Architect** | 方案架构设计 | 架构师/Tech Lead | 拿到需求后如何设计技术方案？选型依据是什么？ | **功能**：输出模块划分、Odoo集成点分析、实现方式决策。<br>**产出**：方案架构文档、实现方式决策说明、风险点清单。 |
| 3 | **Document Generator** | 文档生成 | 全体产研 | 如何将分析结果快速生成标准化、可交付的文档？ | **功能**：将输入内容转换为规范格式的 Markdown 文档。<br>**产出**：PRD 文档、技术方案文档、会议纪要等。 |
| 4 | **Flowchart Designer** | 流程图设计 | 产品/研发 | 业务流程、系统流程如何可视化表达？ | **功能**：根据文字描述生成 Mermaid/PlantUML 流程图。<br>**产出**：业务流程图、时序图、状态机图、泳道图。 |
| 5 | **Data Model Designer** | 数据模型设计 | 后端/DBA | 业务实体有哪些？字段如何定义？表关系如何设计？ | **功能**：从需求中提取实体，设计 ER 图和 Django Model。<br>**产出**：ER 图（Mermaid）、Django Model 代码、字段说明表。区分 Odoo 映射模型（managed=False）和 Django 管理模型。 |
| 6 | **API Draft Designer** | 接口草案设计 | 前端/后端 | 需求阶段如何提前定义接口契约，减少联调摩擦？ | **功能**：根据业务场景草拟 RESTful 接口列表和数据结构。<br>**产出**：接口清单、请求/响应示例、错误码定义。 |
| 7 | **Edge Case Hunter** | 边界场景分析 | 产品/测试/研发 | 需求中有哪些被忽略的异常场景和边界条件？ | **功能**：系统性枚举异常路径、并发场景、数据边界。<br>**产出**：边界场景清单、风险提示、建议处理方案。 |
| 8 | **Requirement Reviewer** | 需求评审 | 产品经理 | 如何高效组织需求评审，确保各方对齐？ | **功能**：生成评审议程、检查需求完整性、记录评审结论。<br>**产出**：评审 Checklist、评审纪要模板、待确认问题列表。 |
| 9 | **Requirement-System Correlation** | 需求系统关联检测 | 架构师/研发 | 该需求与现有业务系统有哪些关联？影响哪些模块？可复用哪些能力？ | **功能**：对比需求与现有模块/接口/数据模型，输出影响范围、复用点、冲突与兼容性风险。<br>**产出**：影响模块清单、复用/扩展点说明、关联检测报告。 |
| 10 | **Performance Advisor** | 性能与扩展性 | 架构师/后端 | 方案如何满足性能与扩展性目标？缓存、异步、批量如何决策？ | **功能**：定义性能目标（QPS/延迟）、扩展性假设，输出缓存/异步/批量在方案中的决策。<br>**产出**：性能与扩展性说明、关键路径性能约定。 |

### 🛤️ Skill 组合路径建议

#### 🟢 场景一：全新功能需求（从 0 到 1）
*   **解决问题**：拿到一个全新需求，如何从理解到方案完整落地？
*   **建议组合**：`需求解析` -> `边界场景分析` -> `方案架构设计` -> `数据模型设计` -> `接口草案设计` -> `流程图设计` -> `文档生成`（若有性能/扩展性要求，可增加 `性能与扩展性`）

#### 🔵 场景二：存量功能迭代优化
*   **解决问题**：在已有系统上叠加新需求，如何评估影响范围？
*   **建议组合**：`需求解析` -> `需求系统关联检测` -> `边界场景分析` -> `方案架构设计` -> `接口草案设计` -> `文档生成`

#### 🟣 场景三：需求评审准备
*   **解决问题**：评审前如何确保需求质量，减少评审中的返工？
*   **建议组合**：`需求解析` -> `边界场景分析` -> `需求评审` -> `文档生成`

#### 🟠 场景四：复杂业务建模
*   **解决问题**：业务实体关系复杂，如何清晰建模并对齐各方理解？
*   **建议组合**：`需求解析` -> `数据模型设计` -> `流程图设计` -> `文档生成`
