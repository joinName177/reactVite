# 多维表系统领域参考（Supadone AI）

> 产品专家评审时，用于校验 PM 方案与多维表系统领域的一致性。

## 产品愿景

打造由 AI 驱动的高灵活性、多维度数据协作平台。将电子表格的易用性与数据库结构化能力相结合，配合自动化工作流，实现数据资产的敏捷管理、深度分析与高效协作。

## 核心模块与层级

### 1. 用户与权限
- 多端登录/注册、找回密码
- 个人中心、账户管理
- **多组织切换**：B 端多租户

### 2. 组织与团队
- 树状/全屏架构图
- 成员邀请、移除、离职、解散
- **权限隔离**：组织-项目 级

### 3. 协作工作台
- 项目列表、新建、版本控制、权限配置
- 空间协作、空间设置、删除空间
- 消息中心

### 4. 多维数据引擎
- **数据源**：MySQL、PostgreSQL、API、Excel/CSV
- **视图联动**：新建视图时关联已有或即时新建数据源
- **全链路**：连通性测试、Schema 推断、字段映射、定时同步
- **字段**：自定义类型、样式、校验、跨表关联
- **多维视图**：表格、看板、卡片、日历、时间线、图表
- **数据处理**：筛选模板、排序、分组统计、AI 辅助录入
- **自动化工作流**：触发器、自动化任务

### 5. 通用规范
- **组件**：抽屉（Drawer）、弹窗（Modal）、步骤条
- **状态**：空页面、搜索无结果、加载中、同步异常等 40+ 边缘场景

## 权限层级

组织 → 空间 → 项目 → 数据项（行列级精细权限）

## 完整用户链路

1. 准入与架构：登录 → 组织切换 → 团队架构 → 权限分配
2. 数据集成：接入数据源 → 测试连接 → 字段映射 → 实例化
3. 协作应用：数据录入（行内/抽屉/表单/AI）→ 多维视图切换 → 工作流触发
4. 决策洞察：图表组件 → 绑定度量 → 看板展示

## 数据源设计原则

- 新增前：连通性测试
- 删除前：依赖检查
- Schema 演进：源端变更时支持同步更新映射
